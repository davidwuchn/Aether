What aether publish does

  1. Builds the Go binary and stamps the version
  2. Copies companion files to ~/.aether/system/
  3. Copies platform wrappers (Claude, OpenCode, Codex) to the hub
  4. Verifies versions match between binary and hub

Excluded (never leaves your machine)

  • .aether/data/ (colony state) • .aether/dreams/ (notes) • .aether/oracle/ (research) • .aether/locks/

  1. Layer 1 (The Engine) - The Go code. Compiled into a binary.
  2. Layer 2 (The Warehouse) - Local cache at ~/.aether/. The middleman between source and destination.
  3. Layer 3 (The Storefronts) - Files that Claude, OpenCode, and Codex read in each project.

You edit in the source repo, publish to the hub, and the hub distributes everywhere.
