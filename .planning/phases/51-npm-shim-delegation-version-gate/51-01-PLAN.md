# Plan 51-01: npm Shim Delegation with Version Gate

## Objective

Guard the YAML wiring behind a version check — the update flow should only swap in
the Go-wired YAML files if the Go binary is confirmed working. This prevents the
"half-updated" state where YAML says `aether` but only bash `.aether/aether-utils.sh` exists.

Also: when a user runs any `aether` command, if the Go binary is available and version matches,
delegate to it instead of running the Node.js CLI. If not available or version mismatches, fall back to Node.js.

## Context

- Phase 48 produced goreleaser platform binaries on GitHub Releases
- Phase 49 created the binary downloader (`bin/lib/binary-downloader.js`)
- Phase 50 added `refreshBinary()` to the update flow
- Currently `bin/cli.js` is a pure Node.js CLI using Commander.js
- The Go binary lives at `~/.aether/bin/aether`
- YAML commands in `.claude/commands/ant/` currently reference shell scripts
- The Go-wired YAML commands would call the Go binary directly instead

## Requirements

- **GATE-01**: Before routing to Go binary, check: binary exists at expected path, is executable,
  and reports a version matching the npm package version
- **GATE-02**: Version comparison works without external npm dependencies (custom semver logic)
- **SHM-01**: User's `aether` command delegates to the Go binary when the version gate passes,
  falling back to Node.js CLI when it does not
- **SHM-02**: Commands that must run in Node.js (install, update, setupHub) always stay in Node.js
  regardless of binary availability

## Tasks

### Task 1: Create version gate module

**What:** Create `bin/lib/version-gate.js` with:

1. `checkBinary()` — returns `{ available: boolean, path: string, version: string | null }`
   - Checks `~/.aether/bin/aether` exists
   - Checks it's executable (`fs.accessSync(path, fs.constants.X_OK)`)
   - Runs `aether version --short` to get the binary's reported version
   - Compares against npm package version using custom semver

2. `compareVersions(a, b)` — custom semver comparison, no external deps
   - Handle `v` prefix stripping
   - Compare major.minor.patch numerically
   - Return -1, 0, or 1

3. `NODE_ONLY_COMMANDS` — array of command names that must always run in Node.js:
   `['install', 'update', 'setup', 'setup-hub']`

4. `shouldDelegate(args)` — given process.argv, returns boolean
   - If command is in NODE_ONLY_COMMANDS → false
   - If version gate passes → true
   - Otherwise → false

**Test:** `tests/unit/version-gate.test.js`
- compareVersions: equal, greater, lesser, v-prefix handling, prerelease tags
- checkBinary: missing binary, non-executable, version match, version mismatch
- shouldDelegate: node-only commands stay, other commands delegate when available

### Task 2: Wire delegation into bin/cli.js

**What:** Add a delegation shim at the top of the CLI entry point. Before Commander parses anything:

```js
const { shouldDelegate, getBinaryPath } = require('./lib/version-gate');

if (shouldDelegate(process.argv)) {
  const binaryPath = getBinaryPath();
  const result = child_process.spawnSync(binaryPath, process.argv.slice(2), {
    stdio: 'inherit',
    env: process.env
  });
  process.exit(result.status);
}
```

This runs BEFORE any Commander setup, so it's a pure passthrough. If the gate fails
or the command is node-only, execution falls through to the existing Node.js CLI.

**Where:** `bin/cli.js`, near the top, before `program` setup. Also ensure
`require.main === module` guard wraps the delegation so it only activates when
run as CLI (not when required as a module).

**Test:** Add integration test verifying delegation bypass works:
- Mock spawnSync to verify Go binary is called
- Verify node-only commands don't delegate even when binary is available

### Task 3: YAML wiring version guard

**What:** During `updateRepo()`, when syncing YAML command files, check if the Go binary
is available before swapping in Go-wired YAML. If the binary isn't available, keep the
shell-based YAML files.

The update flow already syncs files from hub. The guard ensures:
- If binary exists and version matches → sync Go-wired YAML
- If binary missing or version mismatch → keep shell-based YAML, log a warning

**Where:** `bin/cli.js` in the `updateRepo()` function, around the file sync section.

**Test:** Verify that YAML sync behavior changes based on binary availability.
