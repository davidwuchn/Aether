**Test cases to implement:**

1. **COLONY_DATA_DIR resolves to colonies/{sanitized-name}/ when colony exists**
   - Setup: Create temp dir, write COLONY_STATE.json with `colony_name: "Test Colony"`
   - Run: `AETHER_ROOT=<tmp> bash aether-utils.sh colony-name`
   - Verify: Colony name returns "Test Colony"
   - Run a subcommand that reads per-colony files (e.g., `pheromone-read`)
   - Verify: It looks in `colonies/test-colony/` directory

2. **COLONY_DATA_DIR falls back to DATA_DIR when no colony exists**
   - Setup: Create temp dir with `.aether/data/` but NO COLONY_STATE.json
   - Run: A subcommand that references per-colony files
   - Verify: It uses `.aether/data/` directly (no colonies/ subdirectory)

3. **Auto-migration moves flat files to colony subdirectory**
   - Setup: Create temp dir with COLONY_STATE.json + flat files (pheromones.json, session.json, flags.json at DATA_DIR root)
   - Run: Any subcommand that triggers `_resolve_colony_data_dir`
   - Verify: Files moved to `colonies/{name}/`, originals gone from root, COLONY_STATE.json still at root

4. **COLONY_STATE.json stays at root after migration**
   - Setup: Same as test 3
   - Run: Migration trigger
   - Verify: `DATA_DIR/COLONY_STATE.json` exists, `COLONY_DATA_DIR/COLONY_STATE.json` does NOT exist

5. **Colony name sanitization handles edge cases**
   - Setup: Write COLONY_STATE.json with various colony_name values:
     - `"My Project (v2)"` -> should sanitize to `my-project-v2`
     - `"Aether Colony"` -> should sanitize to `aether-colony`
     - `"UPPERCASE"` -> should sanitize to `uppercase`
   - Run: `colony-name` subcommand or check directory resolution
   - Verify: Sanitized names are filesystem-safe (no spaces, lowercase, no special chars)

6. **Lock files include colony name tag (acquire_lock_at)**
   - Setup: Create temp dir with colony state, source file-lock.sh
   - Run: `acquire_lock_at` with a colony tag

   - Cleanup: Release lock

7. **Partial migration recovery (files in both locations)**
   - Setup: Create temp dir with COLONY_STATE.json, some files at root AND some in colonies/{name}/
   - Run: Subcommand that triggers migration
   - Verify: Root files moved to colony dir, existing colony dir files preserved, no duplicates
