Create a new file `pkg/agent/task_router.go` containing:

1. **ParseTypeHint function** -- extracts type hints from task Goal text:
   - Pattern: bracket notation like `[implement]`, `[test]`, `[research]`
   - Use regex `\[([a-z]+)\]` to find the first bracket tag
   - Return the lowercase tag string, or empty string if no bracket found
   - Strip the type hint from the goal text is NOT needed here (just parse, don't modify)

2. **RouteTask function** -- two-pass routing:
   - **Pass 1 (type hint):** Call ParseTypeHint on the task Goal. If non-empty, pass to hintToCaste.
   - **Pass 2 (keyword matching):** If no hint, scan Goal text for keywords:
     - "test", "verify", "assert", "check", "validate" -> CasteWatcher
     - "research", "investigate", "find", "discover", "explore", "analyze" -> CasteScout
     - "design", "architect", "structure" -> CasteArchitect
     - "implement", "create", "build", "add", "write", "fix", "code" -> CasteBuilder
     - Default: CasteBuilder
   - Return the matched Caste

3. **hintToCaste function** -- maps type hint strings to Caste:
   - "implement", "build", "code", "fix" -> CasteBuilder
   - "test", "verify", "validate", "check" -> CasteWatcher
   - "research", "investigate", "explore", "analyze" -> CasteScout
   - "design", "architect", "plan" -> CasteArchitect
   - "review" -> CasteScout (scout does investigation/review)
   - Default -> CasteBuilder
