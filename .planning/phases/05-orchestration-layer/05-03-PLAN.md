   ```go
       registry *Registry

   }
   ```

2. **NewPhaseOrchestrator constructor**:
   ```go
   func NewPhaseOrchestrator(registry *Registry, bus *events.Bus, store *storage.Store) *PhaseOrchestrator
   ```

3. **Run method** -- the main orchestration entry point:
   ```go
   func (o *PhaseOrchestrator) Run(ctx context.Context, phase colony.Phase) (*OrchestrationResult, error)
   ```
   Algorithm:
   a. Call `BuildTaskGraph(phase.Tasks)` to decompose. If cycle detected, return error.
   b. Initialize `o.results = make(map[string]*TaskResult)`.
   c. Enter dispatch loop using Kahn's algorithm:
      - Get ready tasks via `graph.Ready()`
      - If no ready tasks and tasks remain, return error (cycle)
      - Create `errgroup.WithContext(ctx)`, call `g.SetLimit(4)` (match pool default)
      - For each ready task, call `g.Go(func() error { return o.dispatchTask(ctx, task, graph) })`
      - After `g.Wait()`, call `graph.Complete()` for successful tasks
      - Repeat until all tasks processed
   d. Build `OrchestrationResult` from collected results.
   e. Return result.

4. **dispatchTask method** -- dispatches a single task to an agent via the registry:
   ```go
   func (o *PhaseOrchestrator) dispatchTask(ctx context.Context, task TaskNode, graph *TaskGraph) error
   ```
   Implementation:
   a. Create a scoped `context.WithTimeout(ctx, 5*time.Minute)` for the task.
   b. Build a topic string from the caste: `fmt.Sprintf("task.%s", task.Caste)`.
   c. Call `o.registry.Match(topic)` to find agents for this caste.
   d. If no agents matched, return `fmt.Errorf("no agent for task %s (caste: %s)", task.ID, task.Caste)`.
   e. Build a scoped event with payload containing ONLY: `task_id`, `goal`, `criteria`, `type_hint`. Do NOT include sibling tasks or the full phase plan (per D-08 agent isolation).
   f. Call `agent[0].Execute(scopedCtx, scopedEvent)`.
   g. Record result in `o.results` under mutex lock:
      - On success: `TaskResult{TaskID: task.ID, AgentName: agent[0].Name(), Caste: task.Caste, Success: true, Duration: elapsed}`
      - On failure: same but `Success: false, Error: err.Error()`
   h. Return nil (individual task failures do not abort sibling tasks).

5. **OrchestrationResult struct**:
   ```go
   type OrchestrationResult struct {
       PhaseID    int           `json:"phase_id"`
       Tasks      []TaskResult  `json:"tasks"`
       Succeeded  int           `json:"succeeded"`
       Failed     int           `json:"failed"`
       DurationMs int64         `json:"duration_ms"`
       Validated  bool          `json:"validated"`
   }
   ```

6. **validateOutput method** -- checks task results against success criteria:
   ```go
   func (o *PhaseOrchestrator) validateOutput(phase colony.Phase, results map[string]*TaskResult) bool
   ```
   Implementation: For each task in the phase, find its result. If the result is successful, mark its criteria as satisfied. Return true only if all tasks succeeded. (Full criteria evaluation is deferred to the autopilot layer; this is a structural check that results exist and succeeded.)

7. **updateState method** -- updates COLONY_STATE.json with orchestrator progress:
   ```go
   func (o *PhaseOrchestrator) updateState(phaseID int, status string) error
   ```
   Implementation: Loads state via `store.LoadJSON("COLONY_STATE.json", &state)`, updates `state.OrchestratorState` fields (Phase, Status, TaskCount, Completed, Failed, UpdatedAt), and saves back via `store.SaveJSON`.
