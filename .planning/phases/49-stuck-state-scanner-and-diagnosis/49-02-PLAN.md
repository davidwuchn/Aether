Create cmd/recover_visuals.go with three exported functions:

1. `renderRecoverDiagnosis(issues []HealthIssue, state colony.ColonyState) string`:
   - Build output using strings.Builder.
   - Start with renderBanner with an appropriate emoji (e.g., "recover" or use the existing commandEmoji pattern) and "Colony Recovery".
   - If state has a goal: print "Goal: {goal}" on next line.
   - Print phase and state info: "Phase {current}/{total} -- {state}".
   - Print renderStageMarker("Diagnosis").
   - If no issues: print "No stuck-state conditions detected. Colony is healthy." in green (if ANSI colors enabled).
   - If issues exist: for each issue, use writeIssueLine() pattern from medic_cmd.go to format with severity color, category, and message. After the issue line, if issue.Fixable is true, print "  Fixable with --apply" in a dimmer color. If not fixable, print the appropriate hint based on category (e.g., dirty_worktree -> "Needs --apply with confirmation", bad_manifest corrupted -> "Needs --force").
   - After all issues: print renderStageMarker("Summary").
   - Print count line: "{N} issues found ({critical} critical, {warning} warning, {info} info)".
   - Count fixable issues and print: "Run aether recover --apply to fix {N} issues automatically." if any fixable.
   - If no fixable issues but issues exist: print "No automatic fixes available. Review issues above."
   - End with renderNextUp suggestion based on most severe issue category (e.g., missing_build_packet -> "Run aether build {phase} to re-dispatch", partial_phase -> "Run aether continue to advance the colony").

2. `renderRecoverJSON(issues []HealthIssue, state colony.ColonyState, duration time.Duration) string`:
   - Build JSON structure:
     ```json
     {
       "timestamp": "<now RFC3339>",
       "goal": "<state.Goal>",
       "phase": state.CurrentPhase,
       "total_phases": len(state.Plan.Phases),
       "state": string(state.State),
       "issues": <issues array as-is>,
       "summary": {
         "critical": <count>,
         "warning": <count>,
         "info": <count>,
         "fixable": <count>
       },
       "scan_duration_ms": duration.Milliseconds()
     }
     ```
   - Use json.MarshalIndent for readable output.
   - Return the JSON string.

3. `recoverExitCode(issues []HealthIssue) int`:
   - If len(issues) == 0: return 0 (healthy).
   - Otherwise: return 1 (issues detected).
   - Simpler than medicExitCode which returns 0/1/2. For recover, any issue means exit 1.

Then modify cmd/recover.go (from Plan 01) to wire the rendering:

- Else: fmt.Fprint(stdout, renderRecoverDiagnosis(issues, state)).
- For exit code: instead of returning nil always, use os.Exit(recoverExitCode(issues)) when issues exist. Since RunE functions cannot directly set exit codes, follow the medic pattern: return a special error or use the cobra SilenceErrors pattern. The simplest approach matching medic: after printing output, if recoverExitCode(issues) != 0, set a package-level flag and check it at the end, or use fmt.Fprint to stderr and return fmt.Errorf("issues detected") -- but the cleanest way matching the medic pattern is to call os.Exit directly after printing. Check how medic does it and follow the same pattern.
