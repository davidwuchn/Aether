Create cmd/recover.go following the medic_cmd.go pattern:

1. Define `recoverCmd` as a `cobra.Command` with Use "recover", Short "Rescue a stuck colony", Long describing the scan-and-fix purpose, Args cobra.NoArgs, RunE runRecover.
2. In init(), call rootCmd.AddCommand(recoverCmd) and register three flags:
   - --apply (bool, default false) "apply fixes for detected issues"
   - --force (bool, default false) "allow destructive repairs"
   - --json (bool, default false) "output structured JSON"
3. Implement `runRecover` to:
   a. Call loadActiveColonyState(). If error, print the colonyStateLoadMessage and return nil (matching medic pattern).
   b. Resolve dataDir from storage.ResolveAetherRoot.
   c. Call performStuckStateScan(dataDir).
   d. For scan-only mode (no --apply): render issues using writeIssueLine pattern from medic, print summary count, print "Run aether recover --apply to fix N issues" if any fixable, return nil.
   e. Store issues for the visuals/rendering plan to consume later (Plan 02 handles the output formatting).

Create cmd/recover_scanner.go containing the orchestrator and all 7 detection functions:

1. `performStuckStateScan(dataDir string) ([]HealthIssue, error)`:
   - Load state via loadActiveColonyState(). If error, return single critical issue.
   - Call all 7 detectors in dependency order and aggregate issues.
   - Order: scanStaleSpawnedWorkers -> scanBadManifest -> scanMissingBuildPacket -> scanPartialPhase -> scanDirtyWorktrees -> scanBrokenSurvey -> scanMissingAgentFiles
   - Return collected issues, nil.

2. `scanMissingBuildPacket(state colony.ColonyState, dataDir string) []HealthIssue` (DETECT-01):
   - Only check if state.State is EXECUTING or BUILT and state.CurrentPhase > 0.
   - Load manifest via loadCodexContinueManifest(state.CurrentPhase).
   - If !manifest.Present OR manifest.Data.PlanOnly == true OR len(manifest.Data.Dispatches) == 0: return critical fixable issue with category "missing_build_packet".

3. `scanStaleSpawnedWorkers(dataDir string) []HealthIssue` (DETECT-02):
   - Read spawn-runs.json from dataDir. Parse with inline struct matching medic_repair.go:627-633 (CurrentRunID, Runs[] with ID, StartedAt, Status).
   - For each run where Status is "active" or "running" and parseTimestamp(StartedAt) is older than 1 hour: count as stale.
   - If stale count > 0: return critical fixable issue with category "stale_spawned", message reporting count and that they exceeded 1-hour threshold.

4. `scanPartialPhase(state colony.ColonyState, dataDir string) []HealthIssue` (DETECT-03):
   - Only check if state.State is EXECUTING and state.CurrentPhase > 0.
   - Load manifest via loadCodexContinueManifest(state.CurrentPhase).
   - If manifest.Present AND manifest has dispatches AND all dispatches are terminal (status "completed" or "failed", none "spawned" or "active"):
     - Check if continue.json exists at build/phase-N/continue.json via os.Stat.
     - If continue.json missing: return warning fixable issue category "partial_phase", message "Build completed but continue not run".
   - If state.BuildStartedAt is nil AND phase status is "in_progress" AND !manifest.Present:
     - Return warning fixable issue category "partial_phase", message "Phase marked in_progress but never built".

5. `scanBadManifest(state colony.ColonyState, dataDir string) []HealthIssue` (DETECT-04):
   - Only check if state.CurrentPhase > 0.
   - Construct manifest path: filepath.Join(dataDir, "build", fmt.Sprintf("phase-%d", state.CurrentPhase), "manifest.json").
   - Try os.Stat first. If missing: return nil (not a bad manifest -- that is the missing build packet detector's job).
   - Read and JSON-decode the file. If JSON parse fails: return critical issue (NOT fixable -- needs --force) category "bad_manifest", message about parse failure.
   - If parsed: check manifest.Phase != state.CurrentPhase -> warning fixable issue "phase field mismatch".
   - If parsed: check manifest.GeneratedAt == "" -> warning fixable issue "empty generated_at".
   - If parsed: check manifest.State == "" -> warning fixable issue "empty state field".
   - If parsed and manifest has dispatches: check each dispatch TaskID against phase plan tasks. For each dispatch whose TaskID does not match any task in the plan: report warning fixable issue "dispatch references non-existent task".

6. `scanDirtyWorktrees(state colony.ColonyState) []HealthIssue` (DETECT-05):
   - Only check if len(state.Worktrees) > 0.
   - Get actual git worktree paths via getGitWorktreePaths().
   - For each worktree entry in state.Worktrees where Status is not "merged" and not "orphaned":
     - If path exists on disk: run git status --porcelain in that directory. If non-empty output: return critical issue (NOT fixable) category "dirty_worktree", message about uncommitted changes at path.
     - If path does not exist on disk but is in state: return warning fixable issue about state-disk mismatch.
   - Call reportOrphanBranches() and for each orphan branch: return warning fixable issue category "dirty_worktree" with branch name.

7. `scanBrokenSurvey(state colony.ColonyState, dataDir string) []HealthIssue` (DETECT-06):
   - Only check if state.TerritorySurveyed is not nil (survey was run).
   - Survey dir: filepath.Join(dataDir, "survey").
   - For each of the 5 survey files ("blueprint", "chambers", "disciplines", "provisions", "pathogens"):
     - Construct path: filepath.Join(surveyDir, name + ".json").
     - If missing: append warning fixable issue category "broken_survey".
     - If present but invalid JSON: append critical fixable issue category "broken_survey".
     - If present but empty (parsed to nil, {}, null, or []): append warning fixable issue category "broken_survey".
   - Return all collected survey issues.

8. `scanMissingAgentFiles() []HealthIssue` (DETECT-07):
   - Define the 3 surfaces to check (matching medic_wrapper.go pattern):
     - .claude/agents/ant/*.md (expected 25)
     - .opencode/agents/*.md (expected 25)
     - .codex/agents/*.toml (expected 25)
   - For each surface: count files using filepath.Glob. If count < expected: return warning fixable issue category "missing_agents" with count and expected.
   - Cross-check hub: if ~/.aether/system/ exists, check corresponding directories. If hub has the files but local does not, note in the message that "files available in hub".
   - Use the package-level constants expectedClaudeAgents, expectedOpenCodeAgents, expectedCodexAgents from medic_wrapper.go (same package, accessible directly).

Key implementation rules:
- All detector functions return []HealthIssue (empty slice = no issues, not nil).
- Use the helper functions from medic_scanner.go directly: issueCritical(), issueWarning(), issueInfo(), fixableIssue().
- Use parseTimestamp() from medic_scanner.go for all timestamp parsing.
- Do NOT shell out to any aether subcommands. All detection is via direct function calls.
- Do NOT duplicate medic's general health scanning. Recover detects stuck-state only.
- Do NOT check JSON schema conformance, data file corruption beyond what is needed for stuck-state detection. That is medic's job.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem -> recover scanner | All detector input comes from filesystem reads (.aether/data/). Corrupted or maliciously crafted files could cause unexpected behavior. |
| Git worktree -> recover scanner | Dirty worktree detection shells out to git. Git output is trusted but git binary path is not validated. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-01 | Tampering | Manifest JSON parsing | mitigate | Use json.Decoder with DisallowUnknownFields or strict parse; reject malformed JSON gracefully |
| T-49-02 | Denial of Service | scanStaleSpawnedWorkers | accept | Spawn-runs.json is small (<1MB typically). No unbounded loops. |
| T-49-03 | Information Disclosure | JSON output (--json flag) | accept | Diagnosis output contains file paths and state info but no secrets. Acceptable for CLI tool. |
| T-49-04 | Elevation of Privilege | Git worktree commands | mitigate | Only run git status --porcelain (read-only). No git operations that modify state. |
</threat_model>
