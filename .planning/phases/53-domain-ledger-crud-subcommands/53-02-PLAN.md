<objective>
Implement four cobra CLI subcommands for domain review ledger CRUD operations.

Purpose: These commands provide the persistent storage layer for review agent findings across phases. Agents call review-ledger-write to persist findings, colony-prime calls review-ledger-summary to inject prior reviews into worker context, and operators use review-ledger-read and review-ledger-resolve for manual inspection and lifecycle management.

Output: cmd/review_ledger.go with four subcommands, cmd/review_ledger_test.go with integration tests.

    **Helper functions (file-private):**

    1. `nextEntryIndex(entries []colony.ReviewLedgerEntry, prefix string, phase int) int` -- scans entries for IDs matching `{prefix}-{phase}-`, parses trailing number, returns max+1. Returns 1 for no matches.

    2. `formatEntryID(prefix string, phase, index int) string` -- returns `fmt.Sprintf("%s-%d-%03d", prefix, phase, index)`.

    3. `computeSummary(entries []colony.ReviewLedgerEntry) colony.ReviewLedgerSummary` -- iterates entries, counts by Status ("open"/"resolved") and by Severity (HIGH/MEDIUM/LOW/INFO). Returns ReviewLedgerSummary.

    **review-ledger-write command:**
    - Flags: --domain (string, required), --phase (int, required, must be > 0), --findings (string, required, JSON array), --agent (string, optional), --agent-name (string, optional), --phase-name (string, optional)
    - Validate domain against validDomains map
    - If --phase is 0, return error "flag --phase is required and must be > 0"
    - If --agent is provided, validate against agentAllowedDomains. If agent not in map, return error. If domain not in agent's allowed list, return error with allowed domains listed.
    - If --agent is empty, skip agent-domain validation (allows CLI manual use)
    - Parse --findings with json.Unmarshal into a slice of anonymous structs with fields: Severity, File, Line, Category, Description, Suggestion
    - Load existing ledger from `reviews/{domain}/ledger.json` using store.LoadJSON. On error (file missing), create empty ReviewLedgerFile with empty entries slice
    - For each finding, compute next index via nextEntryIndex, build ReviewLedgerEntry with Status="open", GeneratedAt=time.Now().UTC().Format(time.RFC3339), Severity as colony.ReviewSeverity(strings.ToUpper(f.Severity))
    - Append entries, recompute summary via computeSummary, save via store.SaveJSON
    - Output via outputOK with "written", "domain", "total", "summary" fields

    **review-ledger-read command:**
    - Flags: --domain (string, required), --phase (int, optional), --status (string, optional)
    - Validate domain against validDomains
    - Load ledger from `reviews/{domain}/ledger.json`. On error, return empty entries + zero summary
    - Filter: if --phase was explicitly set (check cmd.Flags().Changed("phase")), filter entries by phase
    - Filter: if --status is provided, filter entries by status
    - Return filtered entries + summary from the ledger (not recomputed -- summary is of the full ledger, not the filtered subset)
    - Output via outputOK with "entries" and "summary" fields

    **review-ledger-summary command:**
    - No required flags
    - Iterate the 7 known domains in a deterministic order: security, quality, performance, resilience, testing, history, bugs
    - For each domain, try store.LoadJSON("reviews/{domain}/ledger.json"). If missing, skip. If loaded, extract Summary field
    - Collect into array of domain summaries (each with "domain", "total", "open", "resolved", "by_severity")
    - Output via outputOK with "domains" array field

    **review-ledger-resolve command:**
    - Flags: --domain (string, required), --id (string, required)
    - Validate domain against validDomains
    - Load ledger from `reviews/{domain}/ledger.json`. On error, return "ledger not found"
    - Find entry by ID. If not found, return error "entry {id} not found"
    - Set entry.Status = "resolved", entry.ResolvedAt = pointer to time.Now().UTC().Format(time.RFC3339)
    - Recompute summary via computeSummary, save via store.SaveJSON
    - Output via outputOK with "resolved": true, "id": id

    **init() function:**
    - Register flags for all four commands following the midden_cmds.go init() pattern
    - Add all four commands to rootCmd

    **Important implementation details:**
    - Use `cmd.Flags().Changed("phase")` in review-ledger-read to distinguish "not provided" from "phase 0"
    - Ledger path format: `reviews/{domain}/ledger.json` (relative to store basePath)
    - store.SaveJSON creates intermediate directories via MkdirAll in atomicWriteLocked -- no need to pre-create domain directories
    - Maximum findings per write call: cap at 50 entries. If len(findings) > 50, return error.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> command logic | Untrusted input from --domain, --agent, --findings, --id flags |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-03 | Tampering | --findings JSON input | mitigate | json.Unmarshal with strict struct typing rejects unexpected fields; cap at 50 entries per call to prevent memory exhaustion |
| T-53-04 | Tampering | --domain flag path traversal | mitigate | Validate against validDomains map (only 7 known values); reject any domain not in the map before constructing file path |
| T-53-05 | Tampering | Agent writing to wrong domain | mitigate | agentAllowedDomains map validates agent->domain mapping per LEDG-10; returns clear error with allowed domains |
| T-53-06 | Denial of Service | Large findings array | mitigate | Cap findings at 50 entries per write call; return error if exceeded |
| T-53-07 | Information Disclosure | Ledger file contents | accept | Ledger files are local-only under .aether/data/; no network exposure; same threat model as existing pheromones.json and midden.json |
</threat_model>
