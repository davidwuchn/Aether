    - Test 2: ReviewLedgerEntry omitempty -- marshal entry with zero-value optional fields, verify "resolved_at", "agent_name", "phase_name", "file", "line", "category", "suggestion" are absent from JSON output
    - Test 3: ComputeSummary with 5 entries (2 open HIGH, 1 resolved MEDIUM, 1 open LOW, 1 resolved INFO) returns total=5, open=3, resolved=2, by_severity={high:2, medium:1, low:1, info:1}
    - Test 4: ComputeSummary with empty entries returns zero-value summary
    - Test 5: FormatEntryID("sec", 2, 1) returns "sec-2-001", FormatEntryID("qlt", 10, 23) returns "qlt-10-023"
    - Test 6: NextEntryIndex returns 1 for empty entries, 3 when entries contains "sec-2-001" and "sec-2-002"
  </behavior>
  <action>
    Create pkg/colony/review_ledger.go with the following types:

    1. ReviewSeverity type (string alias) with four constants: ReviewSeverityHigh="HIGH", ReviewSeverityMedium="MEDIUM", ReviewSeverityLow="LOW", ReviewSeverityInfo="INFO"

    2. ReviewLedgerEntry struct with these fields and JSON tags:
       - ID string `json:"id"`
       - Phase int `json:"phase"`
       - PhaseName string `json:"phase_name,omitempty"`
       - Agent string `json:"agent"`
       - AgentName string `json:"agent_name,omitempty"`
       - GeneratedAt string `json:"generated_at"`
       - Status string `json:"status"` (values: "open" or "resolved")
       - Severity ReviewSeverity `json:"severity"`
       - File string `json:"file,omitempty"`
       - Line int `json:"line,omitempty"`
       - Category string `json:"category,omitempty"`
       - Description string `json:"description"`
       - Suggestion string `json:"suggestion,omitempty"`
       - ResolvedAt *string `json:"resolved_at,omitempty"`

    3. ReviewLedgerSeverityCounts struct with High, Medium, Low, Info int fields and `json:"high"`, `json:"medium"`, `json:"low"`, `json:"info"` tags

    4. ReviewLedgerSummary struct with Total, Open, Resolved int fields and BySeverity ReviewLedgerSeverityCounts field, JSON tags: `json:"total"`, `json:"open"`, `json:"resolved"`, `json:"by_severity"`

    5. ReviewLedgerFile struct with:
       - Entries []ReviewLedgerEntry `json:"entries"`
       - Summary ReviewLedgerSummary `json:"summary"`

    6. Exported function ComputeSummary(entries []ReviewLedgerEntry) ReviewLedgerSummary:
       - Initialize zero-value summary
       - Set Total = len(entries)
       - For each entry: increment Open or Resolved; increment BySeverity.High/Medium/Low/Info based on entry.Severity
       - Return summary

    7. Exported function FormatEntryID(domainPrefix string, phase int, index int) string:
       - Return fmt.Sprintf("%s-%d-%03d", domainPrefix, phase, index)

    8. Exported function NextEntryIndex(entries []ReviewLedgerEntry, domainPrefix string, phase int) int:
       - Scan entries for matching prefix+phase pattern
       - Return max existing index + 1, or 1 if no matches

    9. Exported constant/map for valid domains:
       - var ValidReviewDomains = map[string]string{"security": "sec", "quality": "qlt", "performance": "prf", "resilience": "res", "testing": "tst", "history": "hst", "bugs": "bug"}

    Package: colony. Imports: "fmt", "strings". No other dependencies needed.
