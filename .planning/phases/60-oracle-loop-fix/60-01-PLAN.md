<objective>
Fix the OpenCode worker callback URL conflation issue.

The OpenCode dispatcher inherits the full parent environment when spawning worker subprocesses. When the parent has a LiteLLM proxy URL set (e.g. OPENAI_API_BASE), the OpenCode subprocess conflates this proxy URL with the ACP server messaging URL. This fix adds a separate environment variable (AETHER_OPENCODE_AGENT_URL) that overrides the ACP endpoint independently.

Purpose: Workers need to reach the correct ACP messaging endpoint, not the LLM proxy.
Output: Modified platform_dispatch.go with env var injection and tests proving the override works.
</objective>
