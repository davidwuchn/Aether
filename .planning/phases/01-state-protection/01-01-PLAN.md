       func (al *AuditLogger) WriteBoundary(cmd string, destructive bool, mutator func(state *colony.ColonyState) (summary string, err error)) error
       ```
       
       Implementation steps inside WriteBoundary:
       a. Read COLONY_STATE.json via store.ReadFile (captures raw bytes for before-state)
       b. Unmarshal into colony.ColonyState
       c. Call mutator callback -- receives *ColonyState, returns summary string and error

       m. If audit append fails, log to stderr but do NOT roll back the state write (per research anti-pattern guidance)

WriteBoundary relies on store.AtomicWrite and store.AppendJSONL each acquiring their own locks sequentially (state write completes, then audit append happens). This avoids deadlock per the research note about separate locks.

    package storage

    import (
        "fmt"
        "path/filepath"
        "strings"
    )

    var protectedPaths = []string{
        "COLONY_STATE.json",
        "session.json",
        "checkpoints/",
        "midden/",
    }

    type BoundaryGuard struct {
        store   *Store
        allowed map[string]bool
    }

    func NewBoundaryGuard(s *Store) *BoundaryGuard { ... }
    func (bg *BoundaryGuard) Allow(path string) { ... }
    func (bg *BoundaryGuard) CheckWrite(path string) error { ... }
    ```

    - `NewBoundaryGuard` initializes with an empty allowed map.
    - `Allow(path)` adds the cleaned path to the allowed map. Use `filepath.Clean(path)` to prevent traversal.
    - `CheckWrite(path)` checks if the cleaned path matches or is under any protected path. If the path is in the allowed map, return nil. Otherwise return error: `boundary: write to protected path %q requires explicit authorization`.
    - Matching logic: `path == pp || strings.HasPrefix(path, pp)` where `pp` is a protected path. This catches both exact matches and subdirectory writes.
