# Phase 44.2: Command Hygiene and Agent Parity - Research

**Researched:** 2026-04-24
**Domain:** Aether slash command naming convention, agent parity, multi-platform documentation alignment
**Confidence:** HIGH

## Summary

This phase has two independent workstreams: (1) fixing the medic agent body parity mismatch between Claude and OpenCode, and (2) renaming all slash commands from colon format (`ant:command`) to hyphen format (`ant-command`) across the entire repository.

The medic parity fix is a 2-line body content sync plus a trailing newline. The Claude version has two additional workflow steps (step 8 about `aether integrity`, and a `--deep` scan explanation) and two expanded repair-safety bullet points reflecting the newer `aether publish` workflow. The OpenCode version has the older simpler guidance. The fix is to bring the OpenCode body in sync with the Claude canonical version.

The colon-to-hyphen rename is a large mechanical change spanning ~341 unique files and ~1,894 individual line references (excluding `.planning/` which has 108 more). The change touches YAML source definitions, 100 wrapper command files (50 Claude + 50 OpenCode), 20 agent definition files (10 Claude + 10 OpenCode), 30 documentation files, Go source code, Go test files, templates, Codex files, CLAUDE.md, AGENTS.md, CHANGELOG.md, and more. The Go binary does NOT register commands using the `ant:` prefix string -- command names are derived from YAML `name` fields and file names only.

**Primary recommendation:** Split into two tasks -- Task 1 for the medic parity fix (trivial), Task 2 for the colon-to-hyphen rename (large mechanical refactor). The rename must be done as a single atomic commit to avoid leaving the codebase in an inconsistent half-renamed state.

### Pattern 2: Colon-to-Hyphen Rename (Mechanical Search-Replace)
**What:** Replace all `/ant:` references with `/ant-` and `ant:` name fields with `ant-`.
**When to use:** Claude Code convention requires lowercase hyphens in command names, not colons. [CITED: blakecrosley.com guide -- "The name field should be lowercase with hyphens"]

**Key insight:** The command *filenames* do NOT change -- files remain `build.md`, `init.md`, etc. Only the `name:` field in frontmatter and all inline text references change from `ant:X` to `ant-X`.

### Anti-Patterns to Avoid

- **Partial rename:** Do NOT rename some commands and leave others. The entire repo must be consistent.
- **Renaming files:** Do NOT rename `.md` files. The filename `build.md` is independent of the command name `ant-build`.
- **Editing generated wrappers directly:** Command wrappers have `<!-- Generated from .aether/commands/*.yaml - DO NOT EDIT DIRECTLY -->` headers. The `TestCommandWrappersReferenceRealYamlSources` test validates these headers. Edit the source YAML and wrapper content, not generated markers.
- **Forgetting packaging mirrors:** `.aether/agents-claude/` must stay byte-identical to `.claude/agents/ant/`. The parity test checks this.

## Common Pitfalls

### Pitfall 1: Missing a Reference Category
**What goes wrong:** Some files still reference `ant:` after the rename, causing inconsistency.
**Why it happens:** The references span 341+ files across 12+ directories. Easy to miss a category.
**How to avoid:** Use the exhaustive inventory below. After rename, run `grep -rn '/ant:' --include='*.md' --include='*.go' --include='*.yaml' --include='*.json' --include='*.toml' | grep -v '.planning/'` and verify zero results.
**Warning signs:** Any `grep` for `ant:` returning results outside `.planning/`.

### Pitfall 2: Breaking the Generated-From Header Test
**What goes wrong:** `TestCommandWrappersReferenceRealYamlSources` fails because the header references `ant:command.yaml` but the file was renamed.
**Why it happens:** The header pattern is `<!-- Generated from .aether/commands/{name}.yaml - DO NOT EDIT DIRECTLY -->`. If the YAML filename changes, the header must match.
**How to avoid:** Do NOT rename YAML files. Only change the `name:` field inside them. The filenames stay as `build.yaml`, `init.yaml`, etc.
**Warning signs:** `TestCommandWrappersReferenceRealYamlSources` fails.

### Pitfall 3: Breaking the Parity Test After Rename
**What goes wrong:** `TestClaudeOpenCodeAgentContentParity` fails because one platform was updated but not the other.
**Why it happens:** The rename touches agent body references (e.g., `/ant:build` inside agent definitions). If only Claude agents are updated, parity breaks.
**How to avoid:** Apply the same rename to both `.claude/agents/ant/` and `.opencode/agents/` simultaneously. Then sync to `.aether/agents-claude/` mirrors.
**Warning signs:** Parity test reports mismatches.

### Pitfall 4: Forgetting Go Production Code
**What goes wrong:** `codex_visuals.go` still renders `/ant:resume` in help text after rename.
**Why it happens:** Go files are easy to overlook when focused on markdown.
**How to avoid:** The only production Go references are in `cmd/codex_visuals.go` lines 339 and 341. Update these.
**Warning signs:** Visual output tests fail.

### Pitfall 5: Planning Directory Contamination
**What goes wrong:** Planning files (.planning/) get renamed, corrupting phase history and audit trail.
**Why it happens:** `grep` finds 108 references in `.planning/`.
**How to avoid:** Explicitly exclude `.planning/` from the rename. These are historical records and should NOT be modified.

## Code Examples

### Medic Parity Fix (Body Extraction and Sync)

The test extracts body content after the second `---` delimiter:

```go
// Source: cmd/codex_e2e_test.go:999
func extractBodyAfterFrontmatter(data []byte) string {
    content := string(data)
    start := strings.Index(content, "---")
    if start == -1 { return content }
    rest := content[start+3:]
    end := strings.Index(rest, "---")
    if end == -1 { return content }
    return strings.TrimSpace(rest[end+3:])
}
```

The fix: Copy body from `.claude/agents/ant/aether-medic.md` (after frontmatter) into `.opencode/agents/aether-medic.md` (after frontmatter), preserving OpenCode frontmatter unchanged.

### Bulk Rename Pattern

```bash
# Replace /ant: with /ant- in all target files (excluding .planning/)
find . -type f \( -name '*.md' -o -name '*.go' -o -name '*.yaml' -o -name '*.json' -o -name '*.toml' \) \
  -not -path './.planning/*' -not -path './node_modules/*' \
  -exec sed -i '' 's|/ant:|/ant-|g' {} +

# Replace name: ant: with name: ant- in YAML frontmatter
find .aether/commands -name '*.yaml' -exec sed -i '' 's|^name: ant:|name: ant-|' {} +

# Replace name: ant: in command wrapper frontmatter
find .claude/commands/ant .opencode/commands/ant -name '*.md' \
  -exec sed -i '' 's|^name: ant:|name: ant-|' {} +
```

**Note:** On macOS, `sed -i ''` requires the empty string argument. Linux uses `sed -i`.
