<objective>
Replace the hardcoded confidence value in `learning-promote-auto` with a recurrence-calibrated formula based on observation_count, so instincts created from frequently-observed learnings start with higher confidence.

Purpose: Currently `learning-promote-auto` creates instincts with fixed confidence 0.6 regardless of how many times a learning has been observed. The observation_count data exists in learning-observations.json but is not used for confidence scoring. The formula `min(0.7 + (count-1)*0.05, 0.9)` means single-observation learnings start at 0.7 and confidence increases with evidence up to 0.9.

Output: Fixed aether-utils.sh with computed confidence, updated playbook confidence guidelines in continue-advance.md and continue-full.md, and unit tests verifying the formula.
</objective>

Specifically, insert this block after line 5360 (`[[ -z "$colony_count" ]] && colony_count=0`) and before the `fi` on line 5360/the threshold check:

```bash
    # LRN-01: Recurrence-calibrated confidence
    # Formula: min(0.7 + (observation_count - 1) * 0.05, 0.9)
    lp_confidence=$(awk -v c="${observation_count:-1}" 'BEGIN {
      v = 0.7 + (c - 1) * 0.05
      if (v > 0.9) v = 0.9
      if (v < 0.7) v = 0.7
      printf "%.2f", v
    }')
```

Note: the `if (v < 0.7) v = 0.7` guard handles edge cases where observation_count might be 0 or empty.

Then at line 5384, change:
```
--confidence 0.6 \
```
To:
```
--confidence "$lp_confidence" \
```

Also update the evidence string at line 5387 to include the computed confidence:
