<objective>
Align the decision-to-pheromone emission format between `context-update decision` (real-time path) and continue-advance Step 2.1b (batch path) so deduplication works reliably.

Purpose: Currently `context-update decision` emits pheromones as `"Decision: X -- Y"` with source `system:decision` and strength 0.65, while Step 2.1b uses `"[decision] X"` with source `auto:decision` and strength 0.6. This format divergence makes dedup fragile and produces inconsistent signals for workers. Aligning to one format ensures dedup catches already-emitted decisions regardless of which path fired first.

Output: Fixed aether-utils.sh, updated playbook commentary in continue-advance.md and continue-full.md, and a unit test verifying dedup works across both paths.

In `.aether/aether-utils.sh`, find the `context-update decision` handler (~line 508). Change the pheromone-write call from:
```
bash "$0" pheromone-write FEEDBACK "Decision: $decision — $rationale" \
  --strength 0.65 \
  --source "system:decision" \
  --reason "Auto-emitted from architectural decision" \
  --ttl "30d" 2>/dev/null || true
```
To:
```
bash "$0" pheromone-write FEEDBACK "[decision] $decision" \
  --strength 0.6 \
  --source "auto:decision" \
  --reason "Auto-emitted from architectural decision" \
  --ttl "30d" 2>/dev/null || true
```

Three changes: (1) content format from `"Decision: X -- Y"` to `"[decision] X"`, (2) strength from 0.65 to 0.6, (3) source from `system:decision` to `auto:decision`.

Note: The rationale is dropped from the pheromone content because Step 2.1b does not include rationale either (decisions extracted from CONTEXT.md table only have the decision text). This alignment ensures `contains()` dedup matches correctly.

In `.aether/docs/command-playbooks/continue-advance.md` Step 2.1b: Update the dedup check commentary paragraph (line ~232, the paragraph starting "Strength is 0.6...") to remove the mention of `system:decision` source since both paths now use `auto:decision`.
