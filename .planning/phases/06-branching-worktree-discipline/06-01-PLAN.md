    Create a new file `cmd/worktree.go`. Do NOT modify cmd/clash.go.

    The file must contain:

    a) Branch name validation (per D-01, D-02, D-03):
    ```go
    var agentBranchRe = regexp.MustCompile(`^phase-[1-9]\d*/[a-z0-9-]+$`)
    var humanBranchPrefixes = []string{"feature/", "fix/", "experiment/", "colony/"}

    func validateBranchName(name string) error {
        // Reject path traversal
        if strings.Contains(name, "..") {
            return fmt.Errorf("invalid branch name %q: path traversal detected", name)
        }
        // Agent track: phase-N/caste-name
        if agentBranchRe.MatchString(name) {
            return nil
        }
        // Human track: prefix/description
        for _, prefix := range humanBranchPrefixes {
            if strings.HasPrefix(name, prefix) && len(name) > len(prefix) {
                return nil
            }
        }
        return fmt.Errorf("invalid branch name %q: must match phase-N/name or use prefix/name (prefix: feature, fix, experiment, colony)", name)
    }
    ```

    b) Path sanitizer (Pitfall 1 from RESEARCH.md):
    ```go
    func sanitizeBranchPath(branch string) string {
        return strings.ReplaceAll(branch, "/", "-")
    }
    ```

    c) Worktree directory base path:
    ```go
    const worktreeBaseDir = ".aether/worktrees"
    ```

    d) ID generation following the generateFlagID pattern:
    ```go
    func generateWorktreeID() string {
        rnd := make([]byte, 4)
        rand.Read(rnd)
        return fmt.Sprintf("wt_%d_%s", time.Now().Unix(), hex.EncodeToString(rnd))
    }
    ```

    e) `worktree-allocate` command (per D-11, D-12):
    - Flags: `--agent` (string, agent name for agent-track), `--phase` (int, phase number for agent-track), `--branch` (string, branch name for human-track)
    - Logic:
      1. Determine the branch name: if --agent and --phase are provided, construct `phase-{N}/{agent}`. If --branch is provided, use it directly. If neither, error.
      2. Validate the branch name via validateBranchName. Reject invalid names outright (Claude's discretion: strict enforcement supports the discipline goal).
      3. Check if branch already exists in COLONY_STATE.json worktrees. If found with non-merged status, error ("worktree already tracked").
      4. Create the worktree directory path: `.aether/worktrees/{sanitized-branch-name}`. Use `os.MkdirAll` for the parent directory.
      5. Run `git worktree add -b {branch} {worktree-path} HEAD`. If this fails (e.g., branch already exists in git), attempt `git worktree add {worktree-path} {branch}` (reusing existing branch).
      6. If git worktree creation succeeds: load COLONY_STATE.json, append new WorktreeEntry with status "allocated", save. If state save fails, rollback: run `git worktree remove {worktree-path} --force`, then error.
      7. Audit log: call `store.AppendJSONL("state-changelog.jsonl", ...)` with action "worktree-allocate", branch, path, timestamp.
      8. Output JSON with ok:true, branch, path, id, status "allocated".
    - Use `context.WithTimeout(context.Background(), GitTimeout)` for all git operations.
    - If store is nil, call outputError and return.

    f) `worktree-list` command:
    - No flags required. Optional `--status` flag to filter by status.
    - Logic:
      1. Load COLONY_STATE.json. If no worktrees, output empty array.
      2. Cross-reference with `git worktree list --porcelain` to detect discrepancies: worktrees in state but not on disk, worktrees on disk but not in state.
      3. Output JSON with ok:true, worktrees array (each entry has all WorktreeEntry fields + optional "on_disk" boolean).
    - Use parseWorktreePaths from cmd/clash.go (it is an unexported function in the same package, accessible directly).

    g) `worktree-orphan-scan` command (per D-09):
    - Optional `--threshold` flag (int, hours, default 48).
    - Logic:
      1. Load COLONY_STATE.json worktrees. Filter to non-merged entries.
      2. For each worktree, check if path exists on disk. If not, mark as "on_disk: false" (stale state entry).
      3. For each worktree on disk, run `git -C {path} log -1 --format=%ct` to get last commit timestamp. Parse as unix epoch, compare to time.Now(). If difference exceeds threshold hours, flag as orphaned.
      4. Also check `git worktree list --porcelain` for on-disk worktrees NOT in COLONY_STATE.json (untracked worktrees).
      5. Update status of orphaned worktrees to "orphaned" in COLONY_STATE.json. Save.
      6. Audit log entries for each status change.
      7. Output JSON with ok:true, orphaned array (worktrees flagged), untracked array (on-disk but not in state).
    - If a worktree has no commits (empty worktree), use the worktree creation time from WorktreeEntry.CreatedAt.

    h) `init()` function: register all three commands with rootCmd, add flags.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> validateBranchName | Branch names from CLI flags cross into git command arguments |
| COLONY_STATE.json -> program | State file is read and trusted; corruption could cause bad behavior |
| git worktree output -> program | Porcelain output from git is parsed and trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | validateBranchName | mitigate | Reject names containing ".." to prevent path traversal; reject names not matching two-track regex |
| T-06-02 | Tampering | worktree-allocate path construction | mitigate | Use constant worktreeBaseDir (".aether/worktrees/") with sanitized branch name; never accept raw user input as directory path |
| T-06-03 | Tampering | git command injection | accept | exec.CommandContext passes arguments directly (not through shell), so branch names cannot inject commands |
| T-06-04 | Tampering | COLONY_STATE.json writes | mitigate | Use store.SaveJSON (atomic write with file locking) per existing pattern; rollback git worktree if state save fails |
| T-06-05 | Denial of Service | worktree-orphan-scan | accept | Runs on-demand only; 48-hour threshold prevents false positives; no auto-delete of orphaned worktrees |
</threat_model>
