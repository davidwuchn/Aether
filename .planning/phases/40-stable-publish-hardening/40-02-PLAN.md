# Plan 02: E2E Publish Verification and Operations Guide Update

## Objective

Prove the publish flow works end-to-end, reproduce the 1.0.19/1.0.20 mismatch scenario and verify it is now caught, and update the operations guide to document `aether publish`.

---

## Context

Phase 40's success criteria require:
1. `aether install --package-dir "$PWD"` sets hub version.json to match source version
2. After publish, `aether version` and `~/.aether/system/version.json` agree
3. Publish fails loudly if binary and hub cannot be synchronized
4. Reproduce the current 1.0.19/1.0.20 mismatch, then prove it's fixed

The 1.0.19/1.0.20 mismatch occurred because the hub version.json was not being updated when the binary version changed. The new `aether publish` command includes verification that catches this.

---

## Task 1: Create E2E publish verification test

### read_first
- cmd/e2e_install_setup_update_test.go (entire file — E2E test patterns)
- cmd/publish_cmd.go (Task 1 from Plan 01)
- cmd/version.go (modified in Task 3 from Plan 01)
- cmd/root.go (resolveVersion, readInstalledHubVersion)

### action
Create `cmd/e2e_publish_version_test.go` with an E2E test that exercises the full publish → verify flow:

1. **TestE2EPublishVersionAgreement**:
   a. Create temp home directory
   b. Create a mock Aether source checkout in temp dir:
      - go.mod with `module github.com/calcosmic/Aether`
      - cmd/aether/main.go (minimal `package main\nfunc main() {}`)
      - .aether/version.json with `{"version":"1.0.20","updated_at":"2026-04-23"}`
      - .aether/workers.md (minimal content)
      - .aether/skills/ (minimal)
   c. Set up a fake hub dir at `tempHome/.aether/system/` with version.json at "1.0.19" (simulating stale hub)
   d. Run publish with --skip-build-binary and --home-dir pointing to temp home
   e. After publish completes:
      - Read hub version.json — should now be "1.0.20"
      - Assert hub version == source version
   f. Verify that the stale 1.0.19 was overwritten

2. **TestE2EVersionCheckFlag**:
   a. Create temp home with hub at version "1.0.20"
   b. Build a test binary with ldflags setting Version to "1.0.20"
   c. Run `version --check` and assert it passes (exit 0)
   d. Modify hub version.json to "1.0.19"
   e. Run `version --check` and assert it fails (non-zero exit) with "version mismatch" in output

### acceptance_criteria
- `go test ./cmd -run TestE2EPublishVersionAgreement -v` passes
- `go test ./cmd -run TestE2EVersionCheckFlag -v` passes
- All E2E tests still pass: `go test ./cmd -run TestE2E -v` exits 0
- `go test ./...` exits 0 (no regressions)
