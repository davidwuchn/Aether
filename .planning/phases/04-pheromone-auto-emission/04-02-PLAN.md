**PHER-01 (Decision auto-emission):**

1. `pheromone-write auto:decision source creates FEEDBACK signal` -- Call pheromone-write with FEEDBACK type, `[decision] Use awk for parsing` content, `--source auto:decision`, `--strength 0.6`, `--ttl 30d`. 2. `auto:decision pheromones appear in colony-prime output` -- Set up colony with a manually-written pheromone (source auto:decision, type FEEDBACK, content "[decision] Use awk for parsing"), then run colony-prime. Verify prompt_section output contains "[decision]" text. This confirms auto-emitted pheromones flow through the existing pheromone-prime pipeline.

3. `auto:decision pheromone deduplication skips existing signals` -- Pre-populate pheromones.json with an active signal having source "auto:decision" and content "[decision] Use awk for parsing". Call pheromone-write again with the same content. Verify only ONE signal with that content exists in pheromones.json (pheromone-write itself doesn't deduplicate -- this test verifies the dedup check pattern works by checking the count).

**PHER-02 (Midden error pattern emission):**

4. `midden-recent-failures returns failures for grouping` -- Set up midden with 4 failures: 3 in category "security" and 1 in category "test". Call midden-recent-failures. Verify count is 4 and failures array contains all entries. Parse output with jq to group by category and verify "security" group has count >= 3.

5. `pheromone-write auto:error source creates REDIRECT signal` -- Call pheromone-write with REDIRECT type, `[error-pattern] Category "security" recurring (3 occurrences)` content, `--source auto:error`, `--strength 0.7`, `--ttl 30d`. Verify pheromones.json contains a signal with `source === "auto:error"`, `type === "REDIRECT"`, and content includes `[error-pattern]`.

6. `auto:error pheromones appear in colony-prime output` -- Set up colony with a manually-written REDIRECT pheromone (source auto:error, content "[error-pattern] Category security recurring (3 occurrences)"). Run colony-prime. Verify prompt_section contains "[error-pattern]".

**PHER-03 (Success criteria recurrence emission):**

7. `success criteria recurrence detection finds matching criteria across phases` -- Set up COLONY_STATE.json with 3 completed phases where 2 of them share identical success criteria text (e.g., "tests pass without regressions"). Use jq to extract and group success criteria, verify the recurring criterion is detected with count >= 2.

8. `pheromone-write auto:success source creates FEEDBACK signal` -- Call pheromone-write with FEEDBACK type, `[success-pattern] "tests pass without regressions" recurs across phases 1, 2` content, `--source auto:success`, `--strength 0.6`, `--ttl 30d`. Verify pheromones.json contains a signal with `source === "auto:success"`, `type === "FEEDBACK"`, and content includes `[success-pattern]`.

9. `auto:success pheromones appear in colony-prime output` -- Set up colony with a manually-written FEEDBACK pheromone (source auto:success, content "[success-pattern] tests pass without regressions recurs"). Run colony-prime. Verify prompt_section contains "[success-pattern]".

**Cross-cutting:**

10. `auto-emitted pheromones are distinguishable from manual pheromones` -- Create three pheromones: one manual FEEDBACK (source "user"), one auto:decision FEEDBACK, one auto:error REDIRECT. Read pheromones.json and verify each has a distinct source field. Run pheromone-prime and verify all three appear in output.

11. `no pheromones emitted when data sources are empty` -- Set up colony with no CONTEXT.md, empty midden, and no completed phases. Verify pheromones.json has 0 signals (confirming the emission blocks don't crash or emit spurious signals when inputs are empty).
