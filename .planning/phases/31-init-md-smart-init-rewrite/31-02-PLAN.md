<task type="auto">
  <name>Task 1: Write integration tests for smart init flow components</name>
  <files>tests/bash/test-init-smart-flow.sh</files>
  <action>
Create `tests/bash/test-init-smart-flow.sh` with integration tests for the bash components used by the smart init flow.

**Test setup pattern** (follow scan-module.sh and queen-charter.test.sh):
- Each test gets an isolated temp directory
- Copy `aether-utils.sh`, `utils/` directory, and `exchange/` directory to temp dir (exchange/ needed for xml-utils.sh sourcing)
- Source aether-utils.sh from the temp dir
- Clean up after each test

**Tests to write:**

1. **test_prompt_assembly_from_scan_data**: Run `init-research` and verify the extracted fields can be assembled into a prompt structure. Verify:
   - `tech_stack.languages` is extractable via jq
   - `tech_stack.frameworks` is extractable via jq
   - `complexity.size` is one of: small, medium, large
   - `survey_status.suggestion.reason` is present or empty
   - All fields have non-null values or appropriate defaults

2. **test_reinit_detection_with_existing_colony**: Create a temp dir with `.aether/data/COLONY_STATE.json` containing a goal. Verify:
   - Reading the file with `jq -r '.goal'` returns the expected goal
   - Re-init mode flag would be set to true

3. **test_reinit_detection_no_existing_colony**: Create a temp dir WITHOUT COLONY_STATE.json. Verify:
   - File check returns "not found"
   - Re-init mode flag would be set to false

4. **test_charter_extraction_from_queen_md**: Create a temp dir with `.aether/QUEEN.md` containing charter entries (Intent, Vision, Governance, Goal with `[charter]` tags and `(Colony: name)` suffixes). Verify:
   - Each charter field is extractable via grep+sed matching the pattern `\[charter\] \*\*{Field}\*\*: {value} (Colony: {name})`
   - The `(Colony: ...)` suffix is stripped, leaving just the value
   - If a charter field is missing, extraction returns empty string (not error)

5. **test_charter_write_after_approval**: In a temp dir with QUEEN.md initialized via `queen-init`, call `charter-write` with specific intent, vision, governance, and goals. Verify:
   - QUEEN.md now contains `[charter] **Intent**:` entry
   - QUEEN.md now contains `[charter] **Vision**:` entry
   - QUEEN.md now contains `[charter] **Governance**:` entry
   - QUEEN.md now contains `[charter] **Goal**:` entry
   - No new `## ` headers were created
   - Calling charter-write again with different values updates entries in-place

6. **test_reinit_preserves_existing_wisdom**: Create a temp dir with QUEEN.md containing both charter entries AND non-charter wisdom entries. Call charter-write. Verify:
   - Charter entries are updated
   - Non-charter wisdom entries are preserved (not removed)

7. **test_scan_graceful_degradation**: In an empty temp directory (no git, no .aether, no source files), run `init-research`. Verify:
   - Returns valid JSON (no errors)
   - `ok` field is true
   - `result` object exists
   - Empty arrays for languages, frameworks
   - Complexity is "small"

**Implementation notes:**
- Use `assert_json_has_field` from test-helpers.sh for top-level field checks
- Use direct `jq -e` for nested path checks (per v2.5 decision: assert_json_has_field only supports top-level)
- Each test function follows the pattern: setup -> action -> assert -> cleanup
- Use `PASS_COUNT` and `FAIL_COUNT` tracking (same as scan-module.sh)
- All bash in the test file must be POSIX-compatible (no bashisms in shebang line -- use `#!/bin/bash` since aether-utils.sh requires bash)
- Capture stderr where needed: `2>&1` for error path tests (per v2.5 decision: json_err writes to stderr)
