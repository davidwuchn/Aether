  <action>
Create cmd/recover_repair.go with the following contents. Follow the medic repair pattern (cmd/medic_repair.go) exactly for backup, atomic write, and result tracking.

1. **performRecoverRepairs(issues []HealthIssue, dataDir string, force bool, jsonMode bool) (*RepairResult, error)**:
   - Call createBackup(dataDir) first. If it fails, return error immediately (no repairs run without backup).
   - Call cleanupOldBackups(backupsDir, 3) to keep only 3 most recent backups.
   - Filter issues to Fixable=true only.
   - Sort by severityOrder (critical first, then warning, then info).
   - Deduplicate by category+message key using a map[string]bool.
   - For each deduplicated issue:
     a. If isDestructiveCategory(issue.Category) returns true AND force is false:
        - If jsonMode is true: skip (record as "skipped: non-interactive mode"). Do NOT prompt.
        - Otherwise: call confirmRepair(issue). If declined: record as skipped with "user declined".
     b. Otherwise: call dispatchRecoverRepair(issue, dataDir, force).
     c. Track result in RepairResult counts (Succeeded, Failed, Skipped).
     d. Call logRepairToTrace(record, dataDir) for audit trail.
   - If any repair Failed (record.Success == false && record.Error != ""), attempt rollback: read the backup manifest to find the backup path, then call restoreFromBackup(backupPath, dataDir) to copy backup files back over the mutated ones.
   - Return the RepairResult.

2. **isDestructiveCategory(category string) bool**:
   Return true only for "dirty_worktree" and "bad_manifest".

3. **confirmRepair(issue HealthIssue) bool**:
   - Write to os.Stderr: "\n  [confirm] {issue.Message} ({issue.File})\n"
   - Write to os.Stderr: "  Apply fix? [y/N]: "
   - Read from bufio.NewReader(os.Stdin).ReadString('\n')
   - Return true only if trimmed, lowercased response is "y" or "yes".
   - On any read error, return false.

4. **restoreFromBackup(backupPath string, dataDir string) error**:
   - Read all entries from backupPath directory.
   - For each entry, copy from backupPath/entry to dataDir/entry (overwriting).
   - Use backupCopyFile and backupCopyDir (already exist in medic_repair.go).
   - This enables atomic rollback: if any repair in the batch fails, all state files are restored.

5. **dispatchRecoverRepair(issue HealthIssue, dataDir string, force bool) RepairRecord**:
   Switch on issue.Category:
   - "missing_build_packet": return repairMissingBuildPacket(issue, dataDir)
   - "stale_spawned": return repairStaleSpawned(issue, dataDir)
   - "partial_phase": return repairPartialPhase(issue, dataDir)
   - "broken_survey": return repairBrokenSurvey(issue, dataDir)
   - "missing_agents": return repairMissingAgentFiles(issue, dataDir)
   - "dirty_worktree": return repairDirtyWorktree(issue, dataDir, force)
   - "bad_manifest": return repairBadManifest(issue, dataDir)
   - default: return RepairRecord with Action="skip", Error="unsupported category"

6. **repairMissingBuildPacket(issue HealthIssue, dataDir string) RepairRecord**:
   - Read COLONY_STATE.json from dataDir.
   - Parse as colony.ColonyState.
   - record.Before = string(state.State).
   - Use colony.Transition(state.State, colony.StateREADY) to validate the transition is legal. If it fails, set record.Error and return.
   - Set state.State = colony.StateREADY.
   - Set state.BuildStartedAt = nil.
   - Marshal and write via atomicWriteFile.
   - record.Action = "reset_to_ready", record.After = string(colony.StateREADY), record.Success = true.

7. **repairStaleSpawned(issue HealthIssue, dataDir string) RepairRecord**:
   - Read spawn-runs.json from dataDir.
   - Parse into struct with CurrentRunID string and Runs []struct{ID, StartedAt, Status string}.
   - For each run where Status is "running" or "active": if parseTimestamp(run.StartedAt) is more than 1 hour ago, set Status to "failed" and increment reset count.
   - If reset > 0: set spawnState.CurrentRunID = "".
   - Marshal and write via atomicWriteFile.
   - record.Action = "reset_stale_spawns", record.After = fmt.Sprintf("reset %d stale runs to failed", reset).

8. **repairPartialPhase(issue HealthIssue, dataDir string) RepairRecord**:
   - Read COLONY_STATE.json, parse as colony.ColonyState.
   - record.Before = string(state.State).
   - Load the manifest via loadCodexContinueManifest(state.CurrentPhase).
   - If manifest.Present and all dispatches are terminal (completed or failed):
     - Use colony.Transition(state.State, colony.StateBUILT).
     - Set state.State = colony.StateBUILT.
     - record.Action = "transition_to_built".
   - Else (no manifest or incomplete dispatches):
     - Find the current phase in state.Plan.Phases.
     - Set its Status to "pending".
     - Use colony.Transition(state.State, colony.StateREADY) and set state.State = colony.StateREADY.
     - Set state.BuildStartedAt = nil.
     - record.Action = "reset_phase_to_pending".
   - Marshal and write via atomicWriteFile.
   - record.Success = true.

9. **repairBrokenSurvey(issue HealthIssue, dataDir string) RepairRecord**:
   - For multi-file atomicity: first update COLONY_STATE.json, then delete broken files.
   - Read COLONY_STATE.json, parse. Set state.TerritorySurveyed = nil.
   - Marshal and write via atomicWriteFile (state update first -- least destructive).
   - Then iterate over surveyFiles (from cmd/survey.go): for each broken survey file in dataDir/survey/, os.Remove it.
   - record.Action = "clear_broken_survey", record.After = fmt.Sprintf("cleared territory_surveyed, removed broken files").

10. **repairMissingAgentFiles(issue HealthIssue, dataDir string) RepairRecord**:
    - Get repoRoot via resolveAetherRoot().
    - Get hubDir via filepath.Join(homeDir(), ".aether", "system").
    - Define 3 surfaces: Claude (.aether/system/claude/agents -> .claude/agents/ant/), OpenCode (.aether/system/opencode/agents -> .opencode/agents/), Codex (.aether/system/codex/agents -> .codex/agents/).
    - For each surface: if hub source dir exists, glob its files, for each file that does NOT exist in repo destination: os.MkdirAll(dest dir), os.WriteFile(dest, hub file content, 0644). Increment copied counter.
    - If copied == 0 and hub dirs don't exist: record.Error = "hub has no agent files -- run `aether update` first".
    - record.Action = "restore_agent_files", record.After = fmt.Sprintf("restored %d agent files from hub", copied).

11. **repairDirtyWorktree(issue HealthIssue, dataDir string, force bool) RepairRecord**:
    - Read COLONY_STATE.json, parse.
    - Sub-type dispatch by checking issue.Message content:
      a. Contains "state-disk mismatch" or "not in git worktree list": remove matching WorktreeEntry from state.Worktrees (filter where wt.Path != issue.File). record.Action = "remove_orphan_worktree_entry".
      b. Contains "uncommitted change": run exec.Command("git", "-C", issue.File, "stash", "--include-untracked"). If fails, record.Error and return. record.Action = "stash_worktree_changes".
      c. Contains "Orphan branch": extract branch name from issue.File, run exec.Command("git", "branch", "-D", branchName). record.Action = "delete_orphan_branch".
    - Marshal state and write via atomicWriteFile.
    - record.Success = true.

12. **repairBadManifest(issue HealthIssue, dataDir string) RepairRecord**:
    - filePath = filepath.Join(dataDir, issue.File).
    - Read the raw file.
    - If the issue.Message contains "parse failed" (JSON corruption): call findLastValidJSON(raw). If nil: os.Remove the file, record.Action = "remove_corrupt_manifest". If recovered: validate with json.Valid, then atomicWriteFile the recovered bytes. record.Action = "recover_manifest_json".
    - If the issue.Message contains "mismatch" or "empty": parse the manifest, fix the specific fields (set Phase to state.CurrentPhase, set GeneratedAt to now, set State to "executing"), marshal and write via atomicWriteFile. record.Action = "repair_manifest_fields".
    - record.Success = true.

Use the same package `cmd` and imports as existing files (encoding/json, fmt, os, path/filepath, strings, time, bufio, github.com/calcosmic/Aether/pkg/colony).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI -> filesystem | Repair functions mutate .aether/data/ files -- untrusted state files could be crafted |
| CLI -> git worktree | repairDirtyWorktree executes git commands in worktree directories |
| CLI -> hub filesystem | repairMissingAgentFiles copies files from ~/.aether/system/ to repo |
| CLI -> stdin | confirmRepair reads user input from stdin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-01 | Tampering | repairDirtyWorktree | mitigate | Validate worktree path is within repo before executing git stash; use exec.Command without shell expansion |
| T-50-02 | Tampering | repairBadManifest | mitigate | Validate recovered JSON with json.Valid before writing; use findLastValidJSON which scans backwards safely |
| T-50-03 | Denial of Service | performRecoverRepairs | accept | Backup creation could fail on full disk -- error returned, no repairs run. Low-value target for a local CLI tool |
| T-50-04 | Information Disclosure | confirmRepair | accept | Writes issue details to stderr -- acceptable for local CLI usage |
| T-50-05 | Tampering | repairMissingAgentFiles | mitigate | Only copies from hub (~/.aether/system/) which is user-owned; does not execute any copied files |
</threat_model>
