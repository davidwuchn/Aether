# Phase 46 Plan 01: Event Bus Core

## Objective

Implement a Go event bus with typed events, in-memory pub/sub via channels, crash-recoverable JSONL persistence, TTL pruning, and topic-based subscription -- replacing the shell's file-based pub/sub in event-bus.sh.

## Context

- Phase 45 (core-storage) provides the `storage.Store` with `AppendJSONL`, `ReadJSONL`, and `AtomicWrite` operations
- The shell event bus (`.aether/utils/event-bus.sh`) implements 4 subcommands: `_event_publish`, `_event_subscribe`, `_event_cleanup`, `_event_replay`
- Events are persisted as JSONL lines in `$COLONY_DATA_DIR/event-bus.jsonl`
- Each event has: id, topic, payload (JSON), source, timestamp, ttl_days, expires_at
- TTL expiration defaults to 30 days; expired events are pruned on load and on schedule
- Topic matching supports exact match and prefix wildcard (e.g., "learning.*")
- The Go event bus must produce structurally identical JSONL output to the shell

**Implementation:**

  - JSON serialization using custom marshal/unmarshal for RFC3339 timestamps matching shell's `2026-03-31T21:14:07Z` format
  - `Config` struct with DefaultTTL (default 30 days), DefaultLimit (default 50), MaxSubscribers per topic

### Task 2: Bus core with in-memory pub/sub and persistence (type="auto")

**Goal:** Implement the Bus type with Publish, Subscribe, and JSONL persistence in `pkg/events/bus.go`.

**Implementation:**
- Create `pkg/events/bus.go` with:
  - `Bus` struct holding: store (*storage.Store), config (Config), mu (sync.RWMutex), subscribers (map[string][]chan Event), closed (bool)
  - `NewBus(store *storage.Store, config Config) *Bus` constructor
  - `Publish(ctx context.Context, topic string, payload json.RawMessage, source string) (*Event, error)` -- generates ID (`evt_{unix}_{4hex}`), sets timestamp and expiry, persists to JSONL via store.AppendJSONL, then broadcasts to all matching subscribers
  - `Subscribe(topicPattern string) (<-chan Event, error)` -- creates buffered channel, registers under pattern, returns read-only channel
  - `Unsubscribe(topicPattern string, ch <-chan Event)` -- removes subscriber, closes channel
  - `Close()` -- marks bus as closed, closes all subscriber channels
  - ID generation matches shell: `evt_$(date +%s)_$(head -c 2 /dev/urandom | od -An -tx1 | tr -d ' \n')`

### Task 3: Query, replay, cleanup, and crash recovery (type="auto")

**Goal:** Implement Subscribe (query from disk), Replay, Cleanup (TTL pruning), and crash recovery in `pkg/events/bus.go`.

**Implementation:**
- Add methods to Bus in `pkg/events/bus.go`:
  - `Query(ctx context.Context, topicPattern string, since time.Time, limit int) ([]Event, error)` -- reads JSONL, filters by topic + not expired + since, applies limit. Returns empty slice (not error) if file doesn't exist.
  - `Replay(ctx context.Context, topic string, since time.Time, limit int) ([]Event, error)` -- reads JSONL, filters by exact topic + not expired + since, sorts by timestamp ascending, applies limit.
  - `Cleanup(ctx context.Context, dryRun bool) (removed int, remaining int, err error)` -- reads JSONL, filters out expired events, rewrites file atomically. Returns counts.
  - `LoadAndReplay(ctx context.Context) error` -- reads all events from JSONL, filters out expired, broadcasts non-expired to matching subscribers (crash recovery). Called on startup.
- All methods handle missing file gracefully (return empty/zero, no error).
