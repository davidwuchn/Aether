In the colony-prime subcommand (starts at line 7451), add a phase learnings extraction and formatting block BETWEEN the context-capsule section (ends line 7631) and the pheromone signals section (starts line 7633). This follows the exact same conditional-section pattern used by QUEEN wisdom and instinct formatting.

The new block must do these things in order:

1. Read current_phase from COLONY_STATE.json:
   ```
   cp_current_phase=$(jq -r '.current_phase // 0' "$DATA_DIR/COLONY_STATE.json" 2>/dev/null || echo "0")
   ```

2. Set max learnings based on compact mode:
   - If cp_compact is true: cp_max_learnings=5
   - If cp_compact is false: cp_max_learnings=15

3. Extract validated claims from previous phases using jq (the simpler recommended approach from research). The jq filter must:
   - Read memory.phase_learnings array (default to empty array if missing)
   - Include entries where phase is a string (handles "inherited") OR phase is numeric and less than current_phase
   - Flatten to individual claims, keeping only those with status == "validated"
   - Retain phase number and phase_name for grouping
   - Cap total claims at cp_max_learnings (take first N after flattening)
   - Use the simpler jq pattern from research:
     ```
     jq -r --argjson current "$cp_current_phase" --argjson max "$cp_max_learnings" '
       [(.memory.phase_learnings // [])[]
        | select((.phase | type) == "string" or (.phase | tonumber) < $current)
        | .phase as $p | .phase_name as $pn
        | .learnings[]
        | select(.status == "validated")
        | {phase: $p, phase_name: $pn, claim: .claim}
       ] | .[:$max]
     ' "$DATA_DIR/COLONY_STATE.json"
     ```
   - Fall back to "[]" on any error

4. Count the extracted claims using jq length

5. If count > 0, format as actionable guidance text (NOT raw JSON). Target format:
   ```
   --- PHASE LEARNINGS (Previous Phase Insights) ---

   Phase 1 (instinct-pipeline):
     - Confidence floor >= 0.7 prevents noise instincts
     - Error patterns should get higher confidence than success patterns

   Inherited:
     - Claude Code global sync works by copying commands to ~/.claude/commands/

   --- END PHASE LEARNINGS ---
   ```

   Use jq to group claims by phase, sort by phase number (inherited first, then ascending), and format each group with a header line and indented bullet points. Use the formatting pattern from the research section "Formatting Pattern".

6. If count > 0, append cp_learning_section to cp_final_prompt (between context-capsule and pheromone signals)

7. If count == 0, do NOT append anything (no empty section headers -- matches the conditional pattern used by QUEEN wisdom at lines 7603-7622)

8. After extracting learnings, update the log_line by appending learning count:
   ```
   cp_log_line="$cp_log_line, $cp_learning_count learnings"
   ```
   This goes AFTER the pheromone-prime extraction (line 7586) but BEFORE the final JSON assembly (line 7639). Only append if cp_learning_count > 0.

IMPORTANT: Use the cp_ prefix for all variables (colony-prime namespace convention used throughout). Do NOT modify pheromone-prime. Do NOT modify build-context.md or build-wave.md (confirmed in Phase 1 that the existing pipeline chain works).
