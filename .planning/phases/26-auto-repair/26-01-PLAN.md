<objective>
Implement the `--fix` repair engine for the Medic. When `aether medic --fix` runs, it scans first (Phase 25 scanners), then attempts to repair all fixable issues in priority order (critical → warning → info). Each repair is logged to trace.jsonl. Destructive repairs require `--force`.

Purpose: Give users one command to diagnose AND fix common colony problems, with full audit trail.
</objective>

         ```go
             Repairs   []RepairRecord

         ```

         ```go
         type RepairRecord struct {
             Category string
             File     string
             Action   string
             Before   string // truncated snippet of state before
             After    string // truncated snippet of state after
             Success  bool
             Error    string // empty if success
         }
         ```

         - Append a TraceEntry to `trace.jsonl` in `.aether/data/`:
           - level: "intervention"
           - topic: "medic.repair"
           - payload: map with {action, file, before, after, success, error}
           - source: "medic"

       - Implement `performRepairs(scanResult *ScannerResult, opts MedicOptions, dataPath string) (*RepairResult, error)`:

         - Filter issues where `Fixable == true`
         - Sort by severity: critical first, then warning, then info
         - For each fixable issue, dispatch to the appropriate repair handler based on Category:
           - "state" → repairStateIssues
           - "pheromone" → repairPheromoneIssues
           - "session" → repairSessionIssues
           - "data" → repairDataIssues
         - For each repair, call `logRepairToTrace`
         - Return RepairResult with counts
       - Add repair dispatcher stubs (implemented in later tasks):
         - `repairStateIssues(issue HealthIssue, opts MedicOptions, dataPath string) RepairRecord`
         - `repairPheromoneIssues(issue HealthIssue, opts MedicOptions, dataPath string) RepairRecord`
         - `repairSessionIssues(issue HealthIssue, opts MedicOptions, dataPath string) RepairRecord`

| Repair engine → colony data files | Writes allowed only with --fix; destructive writes require --force |
| Repair engine → backup directory | Creates backups before any modification |
| Repair engine → trace.jsonl | Append-only logging for audit trail |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Tampering | Repair modifies colony state | mitigate | Backup before repair, AtomicWrite for safe persistence |
| T-26-02 | Denial of Service | Repair corrupts file during write | mitigate | AtomicWrite uses temp file + rename pattern |
| T-26-03 | Information Disclosure | Trace logs contain before/after snippets | accept | Local-only, snippets truncated to 200 chars |

</threat_model>
