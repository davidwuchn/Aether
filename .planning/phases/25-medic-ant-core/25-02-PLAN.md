  <action>
    1. Implement `scanColonyState(fc *fileChecker) []HealthIssue`:
       - Load and parse COLONY_STATE.json:
         - If file missing: critical "COLONY_STATE.json not found"
         - If JSON parse fails: critical "COLONY_STATE.json is corrupted: {parse error}"
       - Validate required fields:
         - `version` must be "3.0" — if missing or different: warning "State version is '{version}', expected '3.0'"
         - `goal` must be non-empty string — if nil/empty: critical "Colony goal is missing"
         - `state` must be valid State constant (IDLE/READY/EXECUTING/BUILT/COMPLETED) — if invalid: critical "Invalid state '{value}'"
         - `scope` must be valid ColonyScope — if invalid: warning "Invalid scope '{value}'"
       - Validate state consistency:
         - If `state` is EXECUTING but `current_phase` is 0: warning "State is EXECUTING but no current phase"
         - If `state` is IDLE but `goal` is set: info "Colony is IDLE but has a goal set"
         - If `paused` is true but `state` is not READY: warning "Paused flag set but state is {state}, not READY"
       - Validate parallel mode:
         - If `parallel_mode` is set, must be valid ParallelMode — if invalid: warning "Invalid parallel_mode '{value}'"
       - Validate worktrees:
         - For each WorktreeEntry, check `status` is valid (allocated/in-progress/merged/orphaned)
         - Count orphaned worktrees — if >0: warning "{N} orphaned worktree entries"
       - Check deprecated fields:
         - If `signals` array is non-empty: warning "Deprecated 'signals' field has {N} entries — should be migrated to pheromones.json"
       - Validate plan structure if present:
         - If plan has phases, each phase must have valid PhaseStatus (pending/ready/in_progress/completed)
         - Each task must have valid TaskStatus (pending/completed/in_progress)
       - Validate events format:
         - For each event string, verify pipe-delimited format (at least 2 segments)
         - If malformed: warning "Event entry malformed: {entry}"
  </action>
  <acceptance_criteria>
    - `scanColonyState` function exists and returns `[]HealthIssue`
    - Detects: missing file, corrupted JSON, missing goal, invalid state, invalid scope, invalid parallel_mode
    - Detects: orphaned worktrees, deprecated signals, malformed events
    - Detects: state/phase inconsistencies
    - All existing tests still pass (`go test ./cmd/...`)
  </acceptance_criteria>
</task>
