<task type="auto">
  <name>Task 1: Add phase transition and iteration management to oracle.sh</name>
  <files>.aether/oracle/oracle.sh</files>
  <action>
Add three pieces of functionality to oracle.sh:

**1. determine_phase function** (add before the main loop, after generate_research_plan):

A bash function that reads state.json and plan.json, then returns the appropriate phase based on structural conditions:

- **survey -> investigate**: When all questions have been touched (iterations_touched non-empty) OR avg confidence >= 25%
- **investigate -> synthesize**: When avg confidence >= 60% OR fewer than 2 questions below 50% confidence
- **synthesize -> verify**: When avg confidence >= 80%
- **verify**: Stays in verify until completion
- Default fallback: "survey"

Use jq for all metric calculations. The function signature: `determine_phase "$STATE_FILE" "$PLAN_FILE"` and it echoes the new phase name.

**2. build_oracle_prompt function** (add after determine_phase):

A bash function that reads state.json phase field and constructs the complete prompt by:
1. Emitting a phase-specific directive block (10-20 lines) based on current phase
2. Then emitting the contents of oracle.md

Phase directives (use heredoc strings):
- **survey**: "Cast a wide net -- get initial findings for every open question. Target untouched questions first. Aim for 20-40% confidence per question. List all discovered unknowns in gaps.md."
- **investigate**: "Target the lowest-confidence question and go DEEP. You MUST reference existing findings in synthesis.md and ADD NEW information, not restate it. Aim to push confidence above 70%. Update gaps.md with specific remaining unknowns."
- **synthesize**: "Read ALL findings in synthesis.md before doing anything. Identify connections, patterns, and contradictions ACROSS questions. Consolidate redundant findings. Resolve contradictions. Push overall confidence toward the target."
- **verify**: "Focus on claims in gaps.md contradictions section. Cross-reference key findings with additional sources. Confirm or correct confidence scores. Mark well-supported questions as answered with 90%+ confidence. Final gaps.md should contain only genuinely unresolvable unknowns."

The function signature: `build_oracle_prompt "$STATE_FILE" "$SCRIPT_DIR/oracle.md"` and it outputs the combined prompt to stdout.

**3. Update the main loop** (modify existing loop body):

After the AI call completes and after the existing jq validation checks, add:

a. Iteration counter increment:
```bash
ITER_TS=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
jq --arg ts "$ITER_TS" '.iteration += 1 | .last_updated = $ts' "$STATE_FILE" > "$STATE_FILE.tmp" && mv "$STATE_FILE.tmp" "$STATE_FILE"
```

b. Phase transition check:
```bash
NEW_PHASE=$(determine_phase "$STATE_FILE" "$PLAN_FILE")
CURRENT_PHASE=$(jq -r '.phase' "$STATE_FILE")
if [ "$NEW_PHASE" != "$CURRENT_PHASE" ]; then
  echo "  Phase transition: $CURRENT_PHASE -> $NEW_PHASE"
  jq --arg phase "$NEW_PHASE" '.phase = $phase' "$STATE_FILE" > "$STATE_FILE.tmp" && mv "$STATE_FILE.tmp" "$STATE_FILE"
fi
```

c. Replace the direct pipe of oracle.md to the AI CLI:
Change `OUTPUT=$($AI_CMD < "$SCRIPT_DIR/oracle.md" 2>&1 | tee /dev/stderr) || true`
to `OUTPUT=$(build_oracle_prompt "$STATE_FILE" "$SCRIPT_DIR/oracle.md" | $AI_CMD 2>&1 | tee /dev/stderr) || true`

The sequence per iteration should be: build prompt (reads current phase) -> AI runs -> jq validation -> increment iteration -> check/update phase -> regenerate research-plan.md -> check completion. This ensures the NEXT iteration gets the potentially new phase.
