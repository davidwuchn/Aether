# Plan 30-01: Medic Worker Integration

## Goal
Make the Medic a first-class colony worker that auto-spawns during continue-gates
when health issues are detected, and whose findings are injected into colony-prime context.

## Context
Phases 25-29 built: medic scanner, auto-repair, medic skill, ceremony checks,
trace diagnostics. The Medic caste identity already exists in `codex_visuals.go`
(emoji `🩹`, color `96`, label `Medic`). The agent definition exists at
`.claude/agents/ant/aether-medic.md`. What's missing is:
1. Auto-spawn logic during continue-gates
2. Health issue injection into colony-prime context

## Tasks

### Task 1: Auto-spawn logic (`cmd/medic_auto_spawn.go`)
New file with:
- `shouldAutoSpawnMedic(statePath string) (bool, string)` — checks:
  - Stale session (session.json last modified >24h ago)
  - Corrupted state (COLONY_STATE.json has critical health issues)
  - Critical blocker flag exists in flags.json
- `renderMedicAutoSpawnVisual(reason string, name string) string` — visual output
- Uses existing scanner (`performHealthScan`) to detect critical issues

### Task 2: Integrate into continue-gates playbook
Add Step 1.11 (after existing quality gates) in `continue-gates.md`:
- Run `shouldAutoSpawnMedic` check
- If triggered: spawn Medic agent via Agent tool with health findings as context
- Medic produces structured report; critical findings block phase advancement
- Follows same pattern as Gatekeeper (Step 1.8) and Auditor (Step 1.9)

### Task 3: Health context injection into colony-prime
In `cmd/colony_prime_context.go`:
- Add a "medic_health" section builder
- Reads latest medic scan results (if any) from `.aether/data/medic-last-scan.json`
- If critical issues found, inject as high-priority context section
- Section is protected (trim-resistant) when critical issues exist

### Task 4: Persist scan results
After `aether medic` runs (in `runMedic`):
- Save scan results to `.aether/data/medic-last-scan.json`
- Enables colony-prime to pick up latest health status

### Task 5: Tests (`cmd/medic_auto_spawn_test.go`)
- Test shouldAutoSpawnMedic with stale session
- Test shouldAutoSpawnMedic with corrupted state
- Test shouldAutoSpawnMedic with critical blocker
- Test shouldAutoSpawnMedic with healthy colony (no spawn)
- Test medic-last-scan.json persistence
- Test colony-prime includes health section when issues exist

## Success Criteria (from R044)
1. Medic caste exists with emoji, color, and label — ALREADY DONE
2. Can be spawned manually (`/ant:medic`) or automatically — auto-spawn added
3. Auto-spawns when stale session, corrupted state, or critical blocker detected
4. Produces structured health report with recommended next steps
5. Integrates into colony-prime context assembly
