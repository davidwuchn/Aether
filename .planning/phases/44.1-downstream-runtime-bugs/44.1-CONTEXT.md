<decisions>
## Implementation Decisions

### Bug 1: False Skills (codex) Count
- countEntriesInDir in cmd/update_cmd.go:450-466 skips directories, so skills-codex/colony/ and skills-codex/domain/ are invisible
- Fix: change to recursive walk counting SKILL.md files, or count directories in skills-codex root (each dir = one skill)
- The check at line 423 uses filepath.Join(hubSystem, "skills-codex") with expectedCodexSkillCount (29)

### Bug 2: Route-Setter Blocked by Fallback + Refresh Guard
- cmd/codex_plan.go:152-154 blocks --refresh when CurrentPhase > 0, even if colony is only READY
- Route-setter won't overwrite existing fallback outputs (ROUTE-SETTER.md, phase-plan.json, phase-research/*)
- Fix must address BOTH:
  a) Allow --refresh when colony state is READY and no build wave has started
  b) Allow route-setter to replace fallback artifacts generated during the same plan run
- The fallback plan from local synthesis should be treated as provisional, not sacred

### Bug 3: Default Scout Timeout Too Low
- cmd/codex_dispatch_contract.go:11 sets planningScoutTimeout = 5 * time.Minute
- In larger repos the scout routinely exceeds 5m before producing usable output
- Fix: raise default to 15m for scout/route-setter, keep surveyor at 5m
- The --worker-timeout flag already allows override, but the default is the problem

### Claude's Discretion
- Exact implementation approach for each fix
- Whether to add a "provisional" marker on fallback planning artifacts
- Test coverage details

</decisions>

<specifics>
## Specific Requirements

1. **Skills count fix:** Change countEntriesInDir to recursively count SKILL.md files (or count subdirectories when the target is a skill tree). Verify the fix by checking `~/.aether/system/skills-codex/` structure.

2. **Refresh guard fix:** Allow --refresh when ALL phases are in pending/ready state and no build output exists. Block only when a phase is in_progress or completed.

3. **Route-setter overwrite fix:** When the plan command detects that ROUTE-SETTER.md, phase-plan.json, or phase-research/* are fallback artifacts from the current run, the route-setter must be allowed to replace them. Mark fallback artifacts as provisional so the route-setter knows they can be overwritten.

4. **Timeout fix:** Change planningScoutTimeout and planningRouteSetterTimeout from 5m to 15m. Keep surveyorDispatchTimeout and continueReviewTimeout at 5m.

5. **Acceptance test:** After all fixes, `aether init "..."; aether plan --worker-timeout 15m` should produce a repo-specific plan from the route-setter, not a generic language/protocol fallback.

</specifics>
