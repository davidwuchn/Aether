# Requirements: Aether v1.3 Maintenance & Pheromone Integration

**Defined:** 2026-03-19
**Core Value:** The pheromone system should be a living system that auto-emits, persists, and influences worker behavior — not just a storage format

## v1 Requirements

Requirements for this maintenance milestone. Each maps to roadmap phases.

### Data Hygiene

- [ ] **DATA-01**: All test data purged from QUEEN.md (25+ junk entries removed, 5 seed entries preserved)
- [ ] **DATA-02**: All test signals removed from pheromones.json (test signal, demo focus, sanity signal, test area entries)
- [ ] **DATA-03**: Test entries removed from constraints.json focus array
- [ ] **DATA-04**: COLONY_STATE.json reset to clean state (stale Electron-to-Xcode goal removed)
- [ ] **DATA-05**: learning-observations.json purged of synthetic test data (11 fake entries)
- [ ] **DATA-06**: Test entries removed from spawn-tree.txt and midden.json
- [ ] **DATA-07**: data-clean command exists for safe ongoing artifact removal with confirmation prompt

### Pheromone Integration

- [ ] **PHER-01**: Signal injection chain verified end-to-end (user emits signal → pheromone-write → colony-prime → prompt_section → worker receives in spawn context)
- [ ] **PHER-02**: Signal lifecycle works correctly (expiration at phase_end, time-based decay, garbage collection of expired signals)
- [ ] **PHER-03**: Agent definitions updated to acknowledge and act on injected pheromone context (at minimum: builder, watcher, scout)
- [ ] **PHER-04**: Auto-emitted signals during builds verified to influence subsequent build phases
- [ ] **PHER-05**: Midden threshold auto-REDIRECT verified with real failure data
- [ ] **PHER-06**: Pheromone decay math tested with known timestamps and edge cases
- [ ] **PHER-07**: Pheromones persist and carry across sessions (survive /clear, available on /ant:resume)

### Learning Pipeline

- [ ] **LRNG-01**: Observation → learning → instinct pipeline validated end-to-end with real (non-test) data
- [ ] **LRNG-02**: Promoted instincts appear in colony-prime output and influence worker prompts
- [ ] **LRNG-03**: Integration test covers full pipeline: memory-capture → learning-observe → threshold met → learning-promote-auto → instinct-create

### Fresh Install & Commands

- [ ] **INST-01**: Fresh install smoke test validates lay-eggs → init → plan → build on clean repo without errors
- [ ] **INST-02**: All 36 slash commands audited for basic functionality (run with minimal args, document failures)
- [ ] **INST-03**: Critical broken commands fixed (any command that errors on valid input)
- [ ] **INST-04**: Pre-publish validation (validate-package.sh) rejects packages containing test artifacts in data files

### Documentation

- [ ] **DOCS-01**: CLAUDE.md updated to match current reality (no references to eliminated features like runtime/)
- [ ] **DOCS-02**: Pheromone documentation accurately describes injection model (colony-prime injects context, workers don't independently read signals)
- [ ] **DOCS-03**: known-issues.md updated (stale FIXED statuses corrected, resolved items removed)
- [ ] **DOCS-04**: README and user-facing docs reflect verified behavior, not aspirational features

### XML Exchange Activation

- [ ] **XML-01**: XML exchange system wired into commands (/ant:export-signals and /ant:import-signals or equivalent)
- [ ] **XML-02**: Pheromone XML export/import works end-to-end (export from one colony, import into another)
- [ ] **XML-03**: XML exchange integrated into pause/seal lifecycle (automatic export on colony seal)

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Architecture

- **ARCH-01**: Split aether-utils.sh into domain-specific modules (state, pheromone, wisdom, spawn)
- **ARCH-02**: Agent Teams migration for inter-worker communication
- **ARCH-03**: Cross-colony wisdom sync using XML exchange

### Observability

- **OBS-01**: Performance monitoring for command execution time, lock wait time
- **OBS-02**: Audit trail for state changes (who/what modified COLONY_STATE.json)
- **OBS-03**: Dead letter queue for failed spawns

### Hardening

- **HARD-01**: Lock acquisition with exponential backoff and jitter
- **HARD-02**: XXE protection in XML processing
- **HARD-03**: Error code standardization (E_* constants everywhere)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Web/TUI dashboard | Aether is a CLI tool; ASCII dashboards work in terminal context |
| New agent types | 22 agents already cover all needed castes; fix integration before adding |
| Complex pheromone decay algorithms | Linear decay with configurable half-lives is sufficient |
| Per-worker model routing | Claude Code Task tool doesn't support per-subagent env vars |
| Real-time inter-worker messaging | Subagents can't communicate mid-execution; Agent Teams is v2 |
