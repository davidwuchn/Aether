Create cmd/learning_cmds.go with 8 commands for the learning proposal pipeline. Read the shell implementations in aether-utils.sh for each command to understand the logic.

**learning-approve-proposals:**
- Use: "learning-approve-proposals", Short: "Approve pending learning proposals"
- Flags: --all (bool, approve all pending), --ids (string, comma-separated list of proposal IDs)
- Implementation: Load learning-observations.json, find proposals with status "proposed", update status to "approved", save.

**learning-defer-proposals:**
- Use: "learning-defer-proposals", Short: "Defer pending learning proposals"
- Flags: --all (bool), --ids (string, comma-separated)
- Implementation: Same as approve but sets status to "deferred".

**learning-display-proposals:**
- Use: "learning-display-proposals", Short: "Display pending learning proposals"
- Implementation: Load learning-observations.json, filter for status "proposed", outputOK with the list.

**learning-extract-fallback:**
- Use: "learning-extract-fallback", Short: "Extract learning with fallback to simpler approach"
- Flags: --category (string), --content (string), --fallback (string)
- Implementation: Try to extract structured learning from content. If extraction fails, use the fallback string as the learning content. Output the extracted learning.

**learning-inject:**
- Use: "learning-inject", Short: "Directly inject a learning observation"
- Flags: --category (required), --content (required), --trust-score (float64, default 0.5), --source (string, default "manual")
- Implementation: Create a new LearningObservation with generated ID and timestamp, append to learning-observations.json, save.

**learning-promote:**
- Use: "learning-promote", Short: "Promote an observation to an instinct"
- Args: cobra.ExactArgs(1) -- the observation ID
- Implementation: Load the observation by ID from learning-observations.json, create an instinct in COLONY_STATE.json instincts array, mark the observation as promoted.

**learning-select-proposals:**
- Use: "learning-select-proposals", Short: "Select specific proposals for interactive review"
- Flags: --category (string, filter by category)
- Implementation: Load learning-observations.json, filter for status "proposed", optionally filter by category, outputOK with list.

**learning-undo-promotions:**
- Use: "learning-undo-promotions", Short: "Revert recent instinct promotions"
- Flags: --count (int, default 1, how many to undo)
- Implementation: Load COLONY_STATE.json, remove the last N instincts from the array, save. Also update the corresponding observations back to "proposed" status in learning-observations.json.

Register all 8 commands in init() via rootCmd.AddCommand. Create cmd/learning_cmds_test.go with basic tests for learning-inject and learning-display-proposals.

- Implementation: Load COLONY_STATE.json, compute a score (0-100) based on: error count (lower=better), midden entries (lower=better), pheromone signal count, instinct confidence average. outputOK with {score, factors}.

- Use: "eternal-store", Short: "Store a high-value signal in eternal memory"
- Flags: --content (required), --category (string, default "general"), --confidence (float64, default 0.9)
- Implementation: Resolve hub path (~/.aether/eternal/), load memory.json, append entry with generated ID, save. Cap at 200 entries with LRU eviction.

**incident-rule-add:**
- Use: "incident-rule-add", Short: "Add a rule to decree, constraint, or gate file"
- Flags: --type (required, one of: decree, constraint, gate), --rule (required, the rule content), --priority (string, default "normal")
- Implementation: Read the appropriate file from .aether/data/ (decree.json, constraints.json, or gates.json), append the rule, save.

**bootstrap-system:**
- Use: "bootstrap-system", Short: "Copy system files from hub to colony"
