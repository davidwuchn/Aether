**Test Group 1: Injection Chain (PHER-01)**

Test 1: "user-emitted FOCUS signal appears in colony-prime prompt_section"
- Setup: Create test colony with empty pheromones
- Action: Call `pheromone-write FOCUS "security review" --strength 0.8 --source user --reason "User directed"` via runAetherUtil
- Action: Call `colony-prime --compact` via runAetherUtil
- Assert: Parse colony-prime JSON output, extract `.result.prompt_section`
- Assert: prompt_section string contains "security review" (the signal content)
- Assert: prompt_section string contains "FOCUS" or "Pay attention" (the signal type label from pheromone-prime formatting)

Test 2: "user-emitted REDIRECT signal appears in colony-prime with HARD CONSTRAINT label"
- Setup: Create test colony with empty pheromones
- Action: Call `pheromone-write REDIRECT "no console.log" --strength 0.9 --source user`
- Action: Call `colony-prime --compact`
- Assert: prompt_section contains "no console.log"
- Assert: prompt_section contains "REDIRECT" or "HARD CONSTRAINT"

Test 3: "multiple signals appear in colony-prime ordered by effective_strength"
- Setup: Create test colony with empty pheromones
- Action: Write FOCUS signal with strength 0.8, then REDIRECT signal with strength 0.9
- Action: Call `colony-prime --compact`
- Assert: prompt_section contains both signal contents
- Assert: REDIRECT appears before FOCUS in the prompt_section (pheromone-prime sorts REDIRECT first, then by strength desc)

Test 4: "decayed signal below 0.1 threshold does NOT appear in colony-prime"
- Setup: Create test colony, write pheromones.json directly (bypass pheromone-write) with a FOCUS signal where created_at = 29 days ago (strength 0.8, so effective ~= 0.027 which is < 0.1)
- Action: Call `colony-prime --compact`
- Assert: prompt_section does NOT contain the decayed signal's content

**Test Group 2: Signal Lifecycle (PHER-02)**

Test 5: "pheromone-expire --phase-end-only expires phase_end signals"
- Setup: Create test colony with 2 signals: one with expires_at = "phase_end", one with expires_at = future ISO timestamp
- Action: Call `pheromone-expire --phase-end-only`
- Assert: Result shows expired_count = 1
- Action: Read pheromones.json, verify the phase_end signal has active: false
- Assert: The future-dated signal still has active: true

Test 6: "pheromone-expire (no flag) expires time-expired signals"
- Setup: Create test colony with 2 signals: one with expires_at = yesterday's ISO timestamp (active: true), one with expires_at = 30 days from now
- Action: Call `pheromone-expire` (no flags)
- Assert: Result shows expired_count = 1
- Assert: The past-dated signal is expired, the future-dated signal remains active

Test 7: "expired signals are archived to midden"
- Setup: Create test colony with 1 signal with expires_at = "phase_end"
- Action: Call `pheromone-expire --phase-end-only`
- Assert: Read midden.json, verify it contains the expired signal's ID
- Assert: midden.json signals array length >= 1

Test 8: "expired signal does not appear in colony-prime after garbage collection"
- Setup: Create test colony with 1 FOCUS signal with expires_at = "phase_end"
- Action: Verify signal appears in colony-prime output first
- Action: Call `pheromone-expire --phase-end-only`
- Action: Call `colony-prime --compact` again
- Assert: prompt_section does NOT contain the expired signal's content (or prompt_section is empty/null)

Implementation notes:
- Use `test.serial()` for all tests (I/O bound, temp directory operations)
- Each test gets its own temp directory via test.beforeEach/test.afterEach
- For tests that need signals with specific created_at timestamps (Test 4), write directly to pheromones.json instead of using pheromone-write (which always sets created_at to now)
- colony-prime output is JSON: `{"ok":true,"result":{"prompt_section":"...","context_capsule":"..."}}`
- pheromone-expire output is JSON: `{"ok":true,"result":{"expired_count":N,"remaining_active":N,"midden_total":N}}`
