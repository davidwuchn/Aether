<objective>
Unify the dual epoch conversion functions in aether-utils.sh and write comprehensive decay math tests.

Purpose: The pheromone system has two different epoch conversion functions (`to_epoch` using 365 days/year and `approx_epoch` using 365.25 days/year) that produce different values for the same timestamp. This causes inconsistent behavior between pheromone-read (which determines what signals workers see) and pheromone-expire (which garbage collects expired signals). A signal could appear active to workers but get expired by the garbage collector, or vice versa. Additionally, zero tests exist for decay math despite it being the core of signal lifecycle.

Output: A single consistent epoch function across all pheromone subcommands, and a test file covering 9 decay math edge cases.
</objective>
