<task type="auto">
  <name>Task 1: Implement data-clean subcommand in aether-utils.sh</name>
  <files>.aether/aether-utils.sh</files>
  <action>
Add a new `data-clean` subcommand to aether-utils.sh. Place it near the other data management subcommands (around the `validate-state` area or at the end of the case statement before the wildcard `*)`).

The subcommand should accept these flags:
- `--dry-run`: Scan and report only, don't modify files (default behavior when called without `--confirm`)
- `--confirm`: Actually perform the cleanup (required to modify files)
- `--json`: Output results as JSON instead of human-readable text

**What it scans for (artifact detection):**

Scan these files in `.aether/data/`:

1. **pheromones.json** — Look for signals where `content` matches test patterns:
   - Case-insensitive match on: `test signal`, `demo focus`, `sanity signal`, `test area`, `test pheromone`
   - Signal IDs containing `test_` or `demo_` prefix

2. **QUEEN.md** — Look for wisdom entries matching test patterns:
   - Lines containing `test_pattern_`, `test_decree_`, `test_evolution_`
   - Content matching: `Test Pattern:`, `TEST DECREE:`, `Test Evolution:`

3. **learning-observations.json** — Look for observations with test colony IDs:
   - `colony_id` matching: `test-colony`, `test`, `different-colony`, `alpha-colony`, `beta-colony`, `gamma-colony`, `c1`, `c2`

4. **midden/midden.json** — Look for entries in `archived_signals` with test-like content:
   - Signal content matching test patterns (same as pheromones)
   - Entries in `failures` with `colony_id` matching test colony patterns

5. **spawn-tree.txt** — Look for lines containing test worker names:
   - Names matching: `TestAnt`, `Test-Worker`, `test-worker`, `Bolt-99`

6. **constraints.json** — Look for focus entries matching test patterns:
   - Focus area content matching: `test area`, `demo`, `sanity`

**Output format (dry-run / no --confirm):**

```
Data Clean — Artifact Scan
==========================

pheromones.json:
  Found: 0 test signals

QUEEN.md:
  Found: 0 test entries

learning-observations.json:
  Found: 0 test observations

midden.json:
  Found: 0 test entries

spawn-tree.txt:
  Found: 0 test worker lines

constraints.json:
  Found: 0 test focus entries

Total artifacts: 0

{if total > 0}
Run with --confirm to remove these artifacts.
{else}
Colony data is clean. No artifacts found.
{fi}
```

**Output format (with --confirm):**

For each file with artifacts:
- pheromones.json: Filter out matching signals, rewrite file
- QUEEN.md: Remove matching lines/blocks, rewrite file
- learning-observations.json: Filter out matching observations, rewrite file
- midden.json: Filter out matching archived signals and test failures, rewrite file
- spawn-tree.txt: Remove matching lines, rewrite file
- constraints.json: Filter out matching focus entries, rewrite file

Use `jq` for JSON file manipulation. Use `grep -v` patterns or sed for text files (QUEEN.md, spawn-tree.txt).

Output after cleanup:
```json
{"ok": true, "removed": {"pheromones": N, "queen": N, "observations": N, "midden": N, "spawn_tree": N, "constraints": N}, "total": N}
```

**Important implementation details:**
- Use the existing file-lock pattern from aether-utils.sh (call `_acquire_lock` / `_release_lock` if modifying files)
- Handle missing files gracefully (file doesn't exist = 0 artifacts, not an error)
- The `--confirm` flag is what makes it destructive; without it, the command is always safe to run

**Create `.claude/commands/ant/data-clean.md`:**

Follow the established slash command pattern (YAML frontmatter with `name` and `description`, then instructions in markdown). The command should:

1. **Step 1: Scan** — Run `bash .aether/aether-utils.sh data-clean --dry-run` and display the human-readable results showing artifact counts per file.

2. **Step 2: Decision gate** — If artifacts found (total > 0), ask the user: "Remove these N artifacts? (yes/no)". If no artifacts found, display "Colony data is clean" and stop.

3. **Step 3: Clean** — If user confirms, run `bash .aether/aether-utils.sh data-clean --confirm` and display the results showing what was removed.

4. **Step 4: Summary** — Show a brief confirmation of what was cleaned and suggest `/ant:status` to verify colony state.

5. **Step 5: Next Up** — Generate the standard Next Up block (same pattern as other commands):
```bash
state=$(jq -r '.state // "IDLE"' .aether/data/COLONY_STATE.json)
current_phase=$(jq -r '.current_phase // 0' .aether/data/COLONY_STATE.json)
total_phases=$(jq -r '.plan.phases | length' .aether/data/COLONY_STATE.json)
bash .aether/aether-utils.sh print-next-up "$state" "$current_phase" "$total_phases"
```

The command does NOT require a colony to be initialized (it should work even without COLONY_STATE.json having a goal, since the data files might have artifacts from a previous colony).
