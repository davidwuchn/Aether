## Replacement Rules

| Shell Pattern | Go Pattern | Notes |
|---------------|------------|-------|
| `bash .aether/aether-utils.sh <cmd>` (no args) | `aether <cmd>` | Direct replacement |
| `bash .aether/aether-utils.sh generate-progress-bar "$var1" "$var2" 20` | `aether generate-progress-bar --current "$var1" --total "$var2" --width 20` | Positional to flags |
| `bash .aether/aether-utils.sh pheromone-write FOCUS "text" --strength 0.8` | `aether pheromone-write --type FOCUS --content "text" --strength 0.8` | Mixed to all-flags |
| `bash .aether/aether-utils.sh print-next-up "$state" "$current_phase" "$total_phases"` | `aether print-next-up` | Go reads state directly, no args needed |
| `bash .aether/aether-utils.sh flag-list --type blocker` | `aether flag-list --type blocker --json` | Add --json when piped to jq |
| `bash .aether/aether-utils.sh version-check` | `aether version-check-cached` | Deprecated command replacement |

## Commands That Read State Directly (No Args in Go)
These Go commands take NoArgs and read COLONY_STATE.json internally. Remove all positional args:
- `print-next-up` (YAML passes state/phase/total, Go reads them)
- `flag-check-blockers` (Go reads from state)
- `milestone-detect` (Go reads from state)
- `memory-metrics` (Go reads from state)
- `colony-vital-signs` (Go reads from state)
- `data-safety-stats` (Go reads from state)
- `pheromone-count` (Go reads from state)
- `load-state` / `unload-state` (Go reads from state)
- `status` (Go reads from state)

## Table Commands That Need --json When Piped to jq
- `flag-list` -- add `--json` when output is piped through `jq`
- `history` -- add `--json` when output is piped through `jq`
- `phase` -- add `--json` when output is piped through `jq`

## pheromone-count Casing Compatibility
The Go binary `pheromone-count` returns UPPERCASE keys: `{FOCUS: N, REDIRECT: N, FEEDBACK: N}`.
The old shell `pheromone-count` returned lowercase keys: `{focus: N, redirect: N, feedback: N}`.
After wiring, any YAML file that pipes pheromone-count output through jq MUST use uppercase
key references. If a YAML file contains `pheromone-count | jq` patterns with lowercase keys,
update them to uppercase. As of plan creation, no YAML files pipe pheromone-count through jq
(they display raw output), but the executor MUST verify this remains true after all edits.
