**Individual ant commands (8):**

Each individual ant command has the same structure:
- `--dry-run` flag (default false)
- Creates the ant via its New* constructor (passing `store` and `bus` where required)
- Calls `ant.Run(ctx, dryRun)`
- Outputs: `{name, success, summary, dry_run}`

1. `curation-sentinel` — `curation.NewSentinel(store).Run(ctx, dryRun)`
2. `curation-nurse` — `curation.NewNurse(store).Run(ctx, dryRun)`
3. `curation-critic` — `curation.NewCritic(store).Run(ctx, dryRun)`
4. `curation-herald` — `curation.NewHerald(store).Run(ctx, dryRun)`
5. `curation-janitor` — `curation.NewJanitor(store, bus).Run(ctx, dryRun)` where `bus = events.NewBus(store, events.DefaultConfig())`
6. `curation-archivist` — `curation.NewArchivist(store).Run(ctx, dryRun)`
7. `curation-librarian` — `curation.NewLibrarian(store, bus).Run(ctx, dryRun)`
8. `curation-scribe` — `curation.NewScribe().Run(ctx, dryRun)`

**Orchestrator command (1):**

9. `curation-run` — flags: `--dry-run` (default false)
   Creates `curation.NewOrchestrator(store, events.NewBus(store, events.DefaultConfig()))`, calls `orchestrator.Run(ctx, dryRun)`.
   Outputs: `{steps: [...], succeeded, failed, skipped, dry_run, duration_ms}`

All commands use the `init()` function pattern to register with `rootCmd.AddCommand(...)`. Each command's Use field matches the shell subcommand name exactly (e.g., `Use: "curation-sentinel"`).
