# Requirements: Aether v2.7 PR Workflow + Stability

**Defined:** 2026-03-30
**Core Value:** Reliably interpret user requests, decompose into executable work, verify outputs, and ship correct work with minimal back-and-forth.

## v1 Requirements

Requirements for the PR Workflow + Stability milestone. Each maps to roadmap phases.

### State Safety

- [ ] **STATE-01**: All COLONY_STATE.json writes use `state-mutate` for atomic mutations — no full-file reconstruction anywhere in the codebase
- [ ] **STATE-02**: Test suite passes cleanly with no state-related failures — COLONY_STATE.json validation tests handle minimal/empty state gracefully

### Pheromone Propagation

- [ ] **PHERO-01**: `pheromone-snapshot-inject` copies active signals from main into a new worktree branch during creation, ensuring REDIRECT and FOCUS signals reach all workers
- [ ] **PHERO-02**: `pheromone-export-branch` exports branch-specific pheromone signals as a snapshot before PR submission
- [ ] **PHERO-03**: `pheromone-merge-back` merges eligible branch signals (user-created, not auto-generated) into main after PR merge, with dedup and conflict resolution

### Midden Collection

- [ ] **MIDD-01**: `midden-collect` ingests failure records from a merged branch into main's midden, with idempotency via merge fingerprints
- [ ] **MIDD-02**: `midden-handle-revert` tags reverted entries rather than deleting them, preserving failure history
- [ ] **MIDD-03**: `midden-cross-pr-analysis` detects systemic failure patterns across multiple PRs

### CI Context Assembly

- [ ] **CI-01**: `pr-context` subcommand produces machine-readable JSON for CI agents, with structured sections (colony state, pheromones, phase context, blockers)
- [ ] **CI-02**: pr-context uses cacheable sources with fallback chains — never hard-fails, returns partial data if any source is unavailable
- [ ] **CI-03**: pr-context respects tighter token budgets (6K normal, 3K compact) for CI environments

### Clash Detection Integration

- [ ] **CLASH-01**: Clash detection runs as a PreToolUse hook during task-as-PR workflow, blocking edits to files modified in other active worktrees
- [ ] **CLASH-02**: Worktree creation copies colony context and injects pheromone snapshots automatically
- [ ] **CLASH-03**: Merge driver resolves package-lock.json conflicts deterministically (keep "ours")

### Release Hygiene

- [ ] **REL-01**: Published npm package excludes all dev artifacts, test data, worktree references, and colony state files
- [ ] **REL-02**: `validate-package.sh` catches any new file types that should be excluded
- [ ] **REL-03**: .npmignore covers all non-distributable paths including new v2.7 modules' test data

### Test Coverage

- [ ] **TEST-01**: All new v2.7 features have integration tests (clash-detect, worktree, merge-driver, pheromone propagation, midden collection, pr-context)
- [ ] **TEST-02**: Test suite achieves 620+ passing tests with zero failures on clean state

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Full Task-as-PR Pipeline

- **PR-01**: Automated PR creation from phase tasks with generated descriptions
- **PR-02**: 5-tier review pipeline (CI Checks → Agent Reviews → Aggregation → Human Gate → Post-Merge)
- **PR-03**: Sequential merge within waves, parallel PRs per wave

### Cross-Colony Coordination

- **XCOL-01**: Hub-level pheromone coordination across multiple active colonies
- **XCOL-02**: Shared blocker registry across colonies working in the same repo

## Out of Scope

| Feature | Reason |
|---------|--------|
| Full task-as-PR automation | Design docs exist; implementation is v2.8 scope |
| Multi-repo colony coordination | Future architecture work |
| Web/TUI dashboard | CLI tool, ASCII dashboards work in terminal |
| LLM-generated CI prompts | Non-deterministic; bash + jq assembly is deterministic |
| Real-time worktree monitoring | tmux-based watch exists; persistent sync is overkill |
